## Abstract
Long-term memory has become increasingly important for LLM agents that operate across extended interactions and evolving task contexts. Recent memory systems have made past experiences more persistent, compact, and retrievable, but retrieval alone does not ensure that a memory provides valid evidence for the current query. When experiences are compressed into reusable fragments, memories from different situations may appear equally relevant if they involve recurring entities or user states. We refer to this failure as context collapse: memories lose the surrounding context needed to judge whether they provide valid evidence for the current query. To address this problem, we propose Contextual Reinstatement for Agentic Memory (RaMem), a framework that turns retrieved memory fragments into contextually verifiable evidence. RaMem operates through four coordinated stages: (i) evidence anchoring grounds each memory in its original episodic conditions, especially event time, mention time, session span, and participants; (ii) recall condition induction derives the evidence conditions implied by the query; (iii) validity-aware retrieval uses these conditions to prioritize context-compatible memories while retaining content-relevant candidates as fallback evidence; and (iv) context-preserved synthesis keeps the selected memories' structured context available to the generator. Experiments on long-term memory benchmarks show that RaMem consistently improves performance over strong memory baselines, with average F1 gains of more than 10% across several backbones.

# RaMem: Contextual Reinstatement for Long-term Agentic Memory — Detailed Summary

## 1. Core Problem and Overall Meaning

- **Background:** Long-term memory is critical for LLM agents that operate across many sessions. Recent systems have made memories more persistent, compact, and retrievable, but retrieval alone does not guarantee that a retrieved memory is valid evidence for the current query.
- **Core problem — context collapse:** When experiences are compressed into reusable fragments, memories from different situations can appear equally relevant because they share recurring entities, topics, or user states, even when only one of them is valid for the current query. The paper calls this *context collapse*: the surrounding episodic context is flattened, so related but context-invalid memories compete as evidence.
- **Why it matters:** Standard memory systems implicitly rely on a “decontextualized evidence assumption” — that a retrieved fragment can serve as evidence simply because it is relevant. The authors argue that a missing property is *contextual verifiability*: a memory should retain enough context for the agent to judge whether it actually supports the current query.
- **High-level goal:** Move from “retrieve relevant fragments” to “retrieve contextually verifiable evidence,” by reinstating the episodic conditions under which a memory is applicable.

## 2. Proposed Methodology

- **Overall framework:** RaMem (Contextual Reinstatement for Agentic Memory) turns retrieved memory fragments into situated evidence using four coordinated stages.

### 2.1 Episodic Memory Anchoring
- Each memory is anchored to the conditions of its original experience, forming an *episodic evidence unit*.
- A memory \(m_i\) consists of content \(x_i\) and episodic context \(e_i\):
  - \(m_i = (x_i, e_i)\)
- The episodic context records coordinates needed for verification:
  - \(e_i = \{\tau^{\text{event}}_i, \tau^{\text{mention}}_i, \tau^{\text{session}}_i, p_i, \ell_i, u_i, z_i\}\)
  - \(\tau^{\text{event}}_i\): time of the described event (when inferable)
  - \(\tau^{\text{mention}}_i\): explicitly mentioned temporal cue (when available)
  - \(\tau^{\text{session}}_i = [s^{\text{start}}_i, s^{\text{end}}_i]\): session span in which the memory was observed
  - \(p_i\): participants; \(\ell_i\): location; \(u_i\): entities; \(z_i\): topic description
- Key design choice: distinguish event time from session time. Session span is treated as a stable episodic coordinate, because many queries depend on when information was mentioned rather than when the event happened.

### 2.2 Recall Condition Induction
- A query is decomposed into an information need and a contextual recall frame:
  - \(q \mapsto (r_q, c_q)\)
- \(r_q\) captures what is being requested; \(c_q\) records the conditions that valid evidence should satisfy:
  - \(c_q = \{\hat{\tau}_q, \hat{p}_q, \hat{\ell}_q, \hat{u}_q, \hat{z}_q\}\)
  - grounded time range, participants, locations, entities, and topic/episode hints.
- This decomposition is necessary because the same type of question may be answerable from multiple episodes.
- Relative temporal expressions (e.g., “last August”) are grounded using the global date range of the interaction history.

### 2.3 Validity-Aware Memory Retrieval
- RaMem uses *selective contextual reinstatement*: context-aware retrieval is activated only when recall conditions can be grounded reliably; otherwise it falls back to content-based retrieval.
- **Content-based candidate retrieval:**
  - Dense semantic retrieval over memory restatements and lexical retrieval over keywords/full text are combined via reciprocal rank fusion:
    - \(C_{\text{content}}(q) = \text{RRF}(C_{\text{sem}}(q), C_{\text{lex}}(q))\)
- **Context-compatible candidate construction:**
  - When recall conditions are grounded, RaMem constructs a candidate list that prioritizes context-compatible memories while retaining content-relevant ones as fallback:
    - \(C_q = \begin{cases} \text{DedupFirst}(C_{\text{ctx}}(q) \oplus C_{\text{content}}(q)), & \text{if recall conditions grounded} \\ C_{\text{content}}(q), & \text{otherwise} \end{cases}\)
  - \(\oplus\) denotes ordered concatenation; DedupFirst keeps the first occurrence of each memory.
- **Temporal compatibility:**
  - For a grounded query range \(\hat{\tau}_q = [t^{\text{start}}_q, t^{\text{end}}_q]\) and memory session span \(\tau^{\text{session}}_i = [s^{\text{start}}_i, s^{\text{end}}_i]\), memory \(m_i\) is temporally compatible if:
    - \(s^{\text{start}}_i \leq t^{\text{end}}_q\) and \(s^{\text{end}}_i \geq t^{\text{start}}_q\)
  - A small buffer can be added around \(\hat{\tau}_q\) to tolerate mismatch between event time and mention time.
- Within the compatible set, memories are ordered by content relevance and temporal proximity.
- An optional reflection step can check whether retrieved evidence is sufficient and issue additional semantic searches.

### 2.4 Context-Preserved Evidence Synthesis
- Selected memories are not reduced back to plain text; their structured context is preserved.
- RaMem merges context-compatible candidates before fallback evidence and applies stable deduplication:
  - \(L_q = \text{DedupFirst}(C^{\text{ctx}}_q \oplus C^{\text{content}}_q)\)
- The final evidence list is the first \(K\) memories:
  - \(C_q = \text{Head}_K(L_q)\)
- Each memory is presented with its content and episodic coordinates (session span, event/mention time, location, persons, entities, topic), allowing the generator to use both remembered content and evidential context.
- The generator is instructed to answer from the provided evidence and return a structured response.

## 3. Experimental Design

- **Datasets and benchmarks:**
  - **LoCoMo**: primary benchmark, multi-session dialogue histories with multi-hop, temporal, open-domain, and single-hop questions.
  - **LongMemEval / LongMemEval-S**: used for additional evaluation with judge-based accuracy.
- **Backbone models:**
  - Closed-source: GPT-4o, GPT-4.1-mini
  - Open-source: Qwen2.5-3B, Qwen3-8B
- **Baselines and compared methods:**
  - Full-history LoCoMo setting, ReadAgent, MemoryBank, MemGPT, A-Mem, LightMem, Mem0, and SimpleMem.
  - SimpleMem is the strongest structured-memory baseline and the main point of comparison; RaMem is built on the same memory construction and indexing backbone but replaces retrieval with contextual reinstatement.
- **Metrics:**
  - Answer quality: token-level F1 and BLEU-1.
  - Retrieval diagnostics: Recall@K, MRR, capped ground-truth rank, hit-only rank, generator failure conditioned on retrieval.
  - Context-collapse diagnostics: distractor-at-rank-1 (D@1), average distractors in top-5, rank gap between ground-truth memory and strongest context distractor.
- **Evaluation subsets:**
  - All verified questions vs temporal-triggered subset (where temporal context is explicitly activated).
  - Context-confusable subset: cases where SimpleMem retrieves at least one strict context distractor in its top-5 results.

## 6. Main Conclusions and Findings

- **RaMem consistently improves answer quality:**
  - On LoCoMo, RaMem achieves the best average F1 across all four backbones.
  - Average F1 gains over SimpleMem exceed 10% on several backbones (e.g., GPT-4o: 39.06 → 51.66; GPT-4.1-mini: 43.24 → 54.23).
- **Retrieval diagnostics improve:**
  - Recall@10 and MRR increase across backbones; capped ground-truth rank decreases.
  - Improvements are especially large on the temporal-triggered subset.
- **Context collapse is frequent and reducible:**
  - SimpleMem ranks a context-invalid distractor first in a large fraction of examples (over 40–50% across backbones).
  - RaMem reduces D@1 and RankGap while improving ground-truth recall.
- **Context shuffle tests confirm mechanism:**
  - Shuffling temporal/session context severely degrades retrieval and answer quality, especially on temporal-triggered questions.
  - Entity and topic shuffling have much smaller effects, indicating that session-level temporal grounding is the dominant operationalized context signal.
- **Efficiency gains:**
  - RaMem with \(K=10\) matches or exceeds SimpleMem with \(K=20\) while using roughly half the retrieved memory budget, indicating better evidence ordering and context efficiency.

## 7. Strengths in Method or Experimental Design

- **Well-motivated problem formulation:**
  - Clearly identifies and formalizes context collapse as a distinct failure mode, separate from simple retrieval miss.
- **Principled design:**
  - The four-stage pipeline directly implements the idea of contextual verifiability: anchor, induce, retrieve validly, and preserve context.
- **Selective and safe use of context:**
  - Contextual reinstatement is activated only when recall conditions can be grounded reliably, reducing the risk of false constraints.
- **Strong and fair baselines:**
  - Comparisons are made against strong structured-memory baselines, especially SimpleMem, using the same memory construction and evaluation protocol.
- **Rich diagnostic evaluation:**
  - Goes beyond final answer quality to analyze retrieval behavior, distractor exposure, and context-collapse-specific metrics.
- **Mechanism-level validation:**
  - Context shuffle tests and ablation studies provide evidence that improvements come from correct content-context binding rather than incidental prompt formatting.

## 8. Weaknesses and Limitations

- **Narrow operationalization of context:**
  - The current implementation primarily relies on session-level temporal grounding, with entity and topic fields playing a secondary role. Richer contextual dimensions (e.g., causal dependencies, affective states, evolving task goals, implicit social relations) are not deeply exploited.
- **Dependence on upstream quality:**
  - RaMem depends on the quality of memory extraction and query analysis. Missing episodic conditions or incorrectly grounded recall conditions can limit performance.
- **Limited benchmark and modality scope:**
  - Experiments are limited to textual conversational memory benchmarks. Multimodal memories, tool-use trajectories, and real-time deployed agents are not evaluated.
- **Design choices and hyperparameters:**
  - The framework introduces choices about how strictly recall conditions should constrain retrieval and how much temporal mismatch to tolerate. The temporal buffer sensitivity shows that performance is not uniformly robust across all settings.
- **Evaluation mostly on static benchmarks:**
  - While results are strong on LoCoMo and LongMemEval, there is limited evidence on how the framework behaves in truly long-horizon, continually evolving deployments.
- **Potential bias and coverage risks:**
  - Context-confusable and temporal-triggered subsets are selected to stress-test the method, but broader distributional robustness and failure modes outside these subsets are less thoroughly characterized.
