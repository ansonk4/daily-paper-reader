## 摘要
大语言模型（LLM）智能体日益依赖记忆系统来维持长期连贯性。近期研究表明，智能体记忆在持续整合过程中会出现退化。然而，现有研究假设记忆来源于无偏的体验。在本研究中，我们识别并形式化了一种新现象：记忆传染——评估器偏差通过智能体记忆的时间间传播。我们指出，当智能体由有偏的评估器进行训练或引导时，其体验会产生偏差；当这些轨迹被存储并整合到记忆后，偏差会传播至未来从同一记忆库中检索的智能体，即使整合是完美的（即oracle条件）。在两种偏差类型（长度偏好、权威偏差）和四个实验阶段中，我们证明了：（1）即使在完美整合（oracle条件）下，记忆传染依然发生，证明偏差输入是传染的充分原因；（2）整合效果因偏差类型而异——能显著削弱长度偏差，但初步放大了权威偏差（单次运行估计），表明存在偏差类型依赖的交互效应；（3）未观察到安全阈值：在污染率低至p=0.2时仍可检测到偏差传播。我们的发现揭示了当前智能体记忆设计中的关键漏洞，并为测量时间间偏差传播提供了形式化工具。

## Abstract
Large Language Model (LLM) agents increasingly rely on memory systems to maintain long-term coherence. Recent work shows that agent memories degrade during continuous consolidation. However, existing research assumes memories are derived from unbiased experiences. In this work, we identify and formalize a novel phenomenon: Memory Contagion -- the cross-temporal propagation of evaluator bias through agent memory. We show that when agents are trained or guided by biased evaluators, their experiences become biased; when these trajectories are stored and consolidated into memory, the bias propagates to future agents retrieving from the same memory store, even when consolidation is perfect (oracle). Across two bias types (length preference, authority bias) and four experimental phases, we demonstrate: (1) Memory Contagion occurs even with perfect consolidation (oracle condition), proving that biased input is a sufficient cause of contagion; (2) Consolidation has opposite effects depending on bias type -- robustly attenuating length bias while preliminarily amplifying authority bias (single-run estimate), suggesting a bias-type-dependent interaction; (3) No observed safe threshold: bias propagation is detected at contamination rates as low as p=0.2. Our findings expose a critical vulnerability in current agent memory designs and provide formal tools for measuring cross-temporal bias propagation.

## 2. 方法论

### 2.1 核心思想

- 构建一个两阶段传播框架：**源智能体（As）**在有偏评估器指导下产生有偏经验，存入记忆库；**目标智能体（At）**从该记忆库检索时继承偏差行为。
- 提出形式化度量指标 $\Gamma_{\text{temporal}}$，用于量化跨时间偏差传播的程度。
- 通过机制分解，将传播路径拆分为**内容路径**（存储了什么）和**检索路径**（如何检索）。

### 2.2 形式化定义

- **有偏评估器模型**：$E_b(\tau) = E_{\text{clean}}(\tau) + \alpha \cdot b(\tau)$，其中 $\alpha$ 为偏差强度，$b(\tau)$ 为偏差函数。
- **两种偏差类型**：
  - **长度偏好**：$b(\tau) = \frac{\text{len}(a) - \mu_L}{\sigma_L}$（偏好更长回答）
  - **权威偏见**：$b(\tau) = \frac{n_{\text{auth}}(a)}{n_{\text{total}}}$（权威标记密度）
- **$\Gamma_{\text{temporal}}$**：使用 Wasserstein 距离（W1）衡量目标智能体在有偏记忆库与无偏记忆库下的行为分布距离。$\Gamma > 0$ 表示偏差传播。
- **机制分解**：$\Gamma_{\text{total}} = \Gamma_{\text{content}} + \Gamma_{\text{retrieval}} + \Gamma_{\text{interaction}}$

### 2.3 算法流程

- 采用拒绝采样（rejection sampling）注入偏差：对每个任务生成4个候选回答，选择评估器分数最高的。
- 记忆整合对比两种策略：**Oracle 整合**（完美合并，无信息损失）和 **LLM 整合**（基于LLM摘要的现实合并）。
- 去偏差方法：通过LLM重写存储的轨迹，移除偏差标记但保留事实内容。

## 6. 主要结论与发现

1. **完美整合无法阻止传染**：Oracle条件下 $\Gamma_A = 13.18$（长度）和 $\Gamma_A = 11.45$（权威，单次），证明偏差输入本身就是传染的充分原因。

2. **整合效果因偏差类型而异**：
   - 长度偏见：LLM整合**衰减**传染84%（$\Gamma_B = 2.03$ vs. $\Gamma_A = 13.18$）
   - 权威偏见：LLM整合**放大**传染47%（$\Gamma_B = 16.80$ vs. $\Gamma_A = 11.45$，单次）
   - 假设解释：全局特征（长度）被压缩，局部标记（权威引用）被浓缩

3. **内容与检索双路径传播**：内容路径（E1）略大于检索路径（E2），交互效应（E3）很小。

4. **无安全阈值**：污染率低至 $p = 0.2$ 时仍可检测到偏差传播。

5. **剂量-响应非线性**：权威偏见在 $p = 0.8$ 达峰后下降，长度偏见单调递增。

6. **去偏差困难**：权威偏见的去偏差效果有限（$E_4 = 4.20$），因权威标记与语义内容纠缠。

7. **权威偏见传播具有领域依赖性**：在合成开放式QA任务上 $\Gamma_A \approx 0$，仅在事实中心任务上传播。
