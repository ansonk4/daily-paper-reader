## 摘要
现有面向大语言模型智能体的记忆基准大多假设为单用户场景，导致面向医院、工作场所、校园和家庭的共享助手鲜有研究。在这些部署场景中，多个主体向公共记忆池写入信息，并以不同角色、范围和关系进行查询，因此记忆质量不仅需要召回能力，还需具备治理能力。我们提出 GateMem，一个面向多主体共享记忆智能体的基准。GateMem 联合评估以下能力：对合法长时域请求的效用及状态更新、跨上下文授权边界的访问控制，以及在显式删除请求后面向智能体的选择性遗忘。该基准涵盖医疗、办公、教育和家庭领域，具备长程多方交互情节、增量记忆注入、隐藏检查点、结构化评判及泄露目标标注。跨多种基线和骨干模型，没有任何方法能同时实现强效用、稳健访问控制和可靠遗忘。长上下文提示通常以高昂的 token 成本获得最佳治理评分，而基于检索和外部记忆的方法虽降低成本，但仍会泄露未经授权或已删除的信息。这些结果表明，当前记忆智能体距离可靠的机构级共享部署仍有很大差距。

## Abstract
Memory benchmarks for LLM agents largely assume single-user settings, leaving shared assistants for hospitals, workplaces, campuses, and households understudied. In these deployments, multiple principals write to a common memory pool and query it under different roles, scopes, and relationships, so memory quality requires governance as well as recall. We introduce GateMem, a benchmark for multi-principal shared-memory agents. GateMem jointly evaluates utility for legitimate long-horizon requests with state updates, access control across contextual authorization boundaries, and agent-facing active forgetting after explicit deletion requests. It spans medical, office, education, and household domains, with long-form multi-party episodes, incremental memory injection, hidden checkpoints, structured judging, and leak-target annotations. Across diverse baselines and backbone models, no method simultaneously achieves strong utility, robust access control, and reliable forgetting. Long-context prompting often yields the best governance score at high token cost, while retrieval-based and external-memory methods reduce cost yet still leak unauthorized or deleted information. These results show current memory agents remain far from reliable shared institutional deployment.

## 1. 核心问题与整体含义

- **研究背景**：现有面向大语言模型（LLM）智能体的记忆基准（如 LoCoMo、LongMemEval、MemoryAgentBench 等）大多假设**单用户、私有记忆**场景，核心目标是最大化召回率。然而，当 LLM 智能体从个人聊天机器人转向**机构级持久助手**（如医院、企业、校园、家庭）时，记忆不再只是“记住什么”，更涉及“谁有权访问”和“删除后是否真正遗忘”。
- **核心问题**：在**多主体共享记忆池**（multi-principal shared-memory）环境中，智能体需要同时满足三个维度的要求：
  - **效用（Utility）**：对合法授权请求提供完整、准确的回答；
  - **访问控制（Access Control）**：对未经授权或越权请求拒绝泄露受保护信息；
  - **主动遗忘（Active Forgetting）**：在用户显式删除请求后，智能体接口层面不再恢复、确认或间接重构已删除信息。
- **整体含义**：论文将记忆评估从单纯的“召回能力”提升为“记忆治理”（Memory Governance）问题，强调三者必须**联合评估**，因为高召回率如果缺乏治理，反而意味着安全漏洞。

## 2. 方法论

### 2.1 核心思想
GateMem 通过构建**多领域、长程、多方交互剧集（episodes）**，在每个剧集的特定时间节点插入**隐藏检查点（hidden checkpoints）**，分别测试效用、访问控制和主动遗忘。评估时，模型仅能看到交互历史、当前请求者身份和相关策略上下文，但看不到查询标签、评分标准或受保护目标。

### 2.2 形式化定义
- **剧集结构**：每个剧集 $e = (S_e, E_e)$，其中 $S_e = (D_e, P_e, R_e, G_e)$ 分别定义领域、主体集合、角色关系与访问规则；$E_e = (\tau_1, \tau_2, \ldots, \tau_T)$ 为时间有序的轮次序列。
- **记忆状态演化**：$M_t^{(e)} = \text{Ingest}(M_{t-1}^{(e)}, \tau_t, S_e)$，即智能体增量摄入每个轮次后更新内部记忆状态。
- **检查点结构**：每个检查点包含可见输入 $c_n = (e_n, t_n, p_n^{req}, x_n)$ 和隐藏标注 $y_n = (q_n, a_n^*, J_n, \Lambda_n)$，其中 $q_n$ 为检查点类别，$a_n^*$ 为期望动作，$J_n$ 为评判规范，$\Lambda_n$ 为受保护泄露目标。
- **响应归一化**：模型响应被归一化为四种动作之一：`answer`、`answer_redacted`、`refuse`、`no_memory`。

### 2.3 评估指标
- **有效效用（U）**：对效用检查点，响应必须使用 `answer` 动作并完整覆盖 $J_n` 中要求的元素。
- **访问控制违规（A）**：对访问控制检查点，若响应泄露 $\Lambda_n` 中的受保护目标，或未采取限制性动作（`refuse` 或 `answer_redacted`），则计为违规。
- **主动遗忘失败（F）**：对主动遗忘检查点，若响应恢复、确认或重构了已删除信息，或未使用 `no_memory` 动作，则计为失败。
- **记忆治理得分（MGS）**：$MGS = U \cdot (1 - A) \cdot (1 - F)$，乘法形式确保三者缺一不可。
- **效率指标**：每检查点的墙钟时间（Sec/ckpt）和 token 数（Tok/ckpt）。

## 3. 实验设计

### 3.1 数据集 / 场景
GateMem 涵盖**四个机构领域**：
- **医疗（Medical）**：21 个剧集，11 种角色，579 个检查点（210 效用 / 192 访问控制 / 177 主动遗忘）。
- **办公（Office）**：17 个剧集，16 种角色，547 个检查点。
- **教育（Education）**：30 个剧集，16 种角色，540 个检查点（均匀分布）。
- **家庭（Household）**：23 个剧集，17 种角色，552 个检查点。

总计 **91 个长程剧集**和 **2,218 个隐藏检查点**。

### 3.2 对比方法（基线）
论文对比了**三种设计家族**共**七个基线配置**：
- **长上下文提示（Long-Context）**：将完整交互历史放入提示。
- **朴素检索增强（RAG-Naive）**：基于 top-k 相似度检索，无策略层。
- **策略感知检索（RAG-Policy）**：在检索中融入请求者和访问策略元数据。
- **A-MEM**：Zettelkasten 启发的结构化笔记记忆系统，支持链接演化。
- **Mem0**：生产级长期记忆系统，支持增量提取与更新。
- **REMem-I / REMEM-S**：基于情景混合图的记忆框架，分别支持迭代推理和单步检索。

### 3.3 骨干模型
使用 **6 种 LLM** 作为骨干：GPT-5.4、Deepseek-V4-Pro、Llama-4-Maverick、GPT-5-mini、GPT-4o-mini、Gemini-2.5-Flash-Lite。

## 6. 主要结论与发现

- **无方法能同时满足三维要求**：没有任何基线在所有骨干和领域上同时实现高效用、低访问控制违规和低主动遗忘失败。
- **长上下文提示治理得分最高但成本高昂**：Long-Context 在多数骨干-领域块中 MGS 最高，但 token 开销大（如办公领域 7.61k tok/ckpt）。
- **策略感知检索改善安全性但牺牲效用**：RAG-Policy 显著降低未授权泄露，但常因过度过滤导致效用下降和过度拒绝上升。
- **显式记忆系统不自动提供治理**：A-MEM、Mem0、REMem 等结构化机制在 MGS 上未持续优于简单基线，说明记忆组织本身不足以解决治理问题。
- **骨干选择影响效用-风险权衡**：GPT-5.4 和 Deepseek-V4-Pro 在治理得分上表现突出，但 Gemini-2.5-Flash-Lite 虽效用高却伴随严重的访问控制和遗忘失败。
- **效率与治理的权衡**：图式记忆系统（如 REMEM）将 token 数降至约 1k/ckpt，但墙钟时间高达 260s/ckpt，远高于 Long-Context 的 4.22s/ckpt。
