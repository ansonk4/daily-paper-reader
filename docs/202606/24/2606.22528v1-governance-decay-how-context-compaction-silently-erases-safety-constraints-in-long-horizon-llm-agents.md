## 摘要
现代大模型智能体日益依赖上下文压缩、摘要或逐出机制，以将长会话控制在词元预算之内。我们发现，这一上下文管理层是一个安全关键的故障面：当上下文中可见的治理约束处于可观测状态时，智能体能够可靠地遵守，但压缩过程可能将其静默移除，导致智能体会话后期执行被禁止的工具调用行为。我们将这种失效模式称为“治理衰减”。我们提出了 ConstraintRot 基准测试，该基准包含长周期智能体场景及确定性的工具调用评分机制，并在七个模型家族中测量了压缩引发的违规行为。在 1,323 个测试片段中，违规率从策略完整保留于上下文时的 0% 升至压缩后的 30%，部分模型甚至达到 59%；当约束在摘要中得以保留时，违规率维持 0%，而一旦被丢弃，违规率则高达 38%。我们进一步研究了一种“压缩-逐出攻击”，即对抗性上下文内容引导摘要器遗漏合法策略，并表明优化后的注入能击败所有被评估模型。最后，我们提出了“约束固定”，一种无需训练的简易缓解方案，通过将治理约束隔离于有损压缩之外，将基准测试中的违规率恢复至 0%。这些结果表明，上下文管理应被视为已部署大模型智能体的一等治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

## 2. 方法论

### 核心思想
- **治理衰减量化**：通过比较策略完整保留时（0%违规）与压缩后（最高59%违规）的违规率差异，量化压缩对安全约束的影响。
- **约束固定（Constraint Pinning）**：提出一种免训练的缓解方案，将治理约束隔离于有损压缩之外，每次压缩后重新注入，确保约束不被丢弃。

### 关键技术细节
- **基准测试（ConstraintRot）**：构建包含9个任务的长周期智能体场景，每个场景包含：(1) 策略回合建立治理约束；(2) 长序列良性任务触发压缩；(3) 违规触发请求。通过确定性工具调用评分（检测工具调用参数中的违规行为）而非LLM评判。
- **压缩驱逐攻击（Compaction-Eviction Attack）**：两种变体：(1) 体积攻击：注入大量内容迫使压缩触发并驱逐策略；(2) 摘要器注入：嵌入指令引导摘要器遗漏策略。优化注入策略可击败所有模型。
- **约束固定机制**：提取治理约束到固定缓冲区，(i) 豁免于压缩，(ii) 每次压缩后逐字重新注入，(iii) 完整性检查确保压缩后上下文仍蕴含固定约束。

### 公式或算法流程
- **治理衰减度量**：Δdecay = v(M, [C(H<t); r]) - v(M, [H<t; r])，其中v(M,H)是模型M在上下文H下执行违规动作的概率，C是压缩算子，r是违规请求。
- **固定约束验证**：压缩后上下文需蕴含固定约束，否则触发警告或重新注入。

## 6. 主要结论与发现

- **治理衰减普遍存在**：压缩后违规率从0%升至30%，部分模型达59%；约束若保留在摘要中违规率保持0%，若被丢弃则达38%。
- **软策略更易受影响**：软组织策略衰减+50点（0%→50%），硬安全规范仅+6点，差异达8.3倍，因硬规范依赖模型内在先验。
- **攻击可武器化**：优化注入策略可击败所有模型（Claude从0%升至65%，DeepSeek达100%）；体积攻击对被动鲁棒模型仍有效（Gemini从4%升至19%）。
- **约束固定有效**：免训练方案将违规率恢复至0%，token开销<0.5%，且不影响任务效用（99%允许动作完成率）。
- **防御边界**：固定机制可被操作者冒充攻击突破（违规率升至17%），需可信带外通道解决。

## 7. 优点

- **问题定义清晰**：首次系统研究上下文压缩对安全治理的影响，提出"治理衰减"概念，填补智能体安全研究空白。
- **基准设计严谨**：ConstraintRot基准包含确定性工具调用评分，避免LLM评判不确定性；任务设计区分软/硬约束，揭示衰减差异。
- **机制验证充分**：通过消融实验（摘要器vs.主体、长度vs.删除）证明衰减源于压缩删除而非通用长度效应；交叉验证（关键词启发式与LLM评判）确保结论可靠。
- **防御方案实用**：约束固定免训练、低成本（<0.5% token开销）、易集成，且通过压力测试揭示其边界（需可信带外通道）。
- **实际验证**：在LangGraph、LangMem、AutoGen、OpenAI Agents SDK等生产框架中复现治理衰减，证明问题现实性。

## 8. 不足与局限

- **系统消息保护局限**：框架保护的系统/开发者消息通道不保护内存或对话传递的治理约束，但论文未完全解决多通道治理的复杂性。

- **操作者冒充攻击未完全解决**：固定机制可被近期上下文中的操作者冒充指令突破，需可信带外通道，但论文未实现该方案。
