## 摘要
用户依赖执行轨迹来观察智能体行为、诊断故障并确保可问责性。这些轨迹包含丰富的程序细节，包括工具调用、中间决策和错误恢复逻辑。然而，这些细节可能暴露私有的程序技能，使下游方法能够在无需访问模型权重或技能文件的情况下恢复关键公式、阈值和策略。为量化这一风险并评估保护措施，我们构建了\textsc{CapTraceBench}基准，包含75个专业的长时域任务和154个跨七个领域精心策划的技能。我们还引入了\textsc{RedAct}（https://github.com/XuShuwenn/RedAct），一个受保护轨迹发布框架，它定位受保护的关键信息，在保留验证者关键证据的同时重写轨迹，并嵌入行为水印以用于下游溯源分析。在代表性的轨迹重用方法中，\textsc{RedAct}将原始轨迹上的归一化技能迁移（NST）从44.7%-67.1%降低到低于无技能基线，同时保留审计证据。其独立的行为水印达到93.6%-100.0%的真实检测率，虚警率最多为1.9%。这些结果将公共智能体轨迹视为安全接口，并表明选择性编辑可以在不删除审计证据的情况下减少程序能力泄露。

## Abstract
Users rely on execution traces to observe agent behavior, diagnose failures, and ensure accountability. These traces contain rich procedural detail, including tool invocations, intermediate decisions, and error-recovery logic. Yet this detail can expose private procedural skills, allowing downstream methods to recover key formulas, thresholds, and strategies without access to model weights or skill files. To quantify this risk and evaluate protection, we construct \textsc{CapTraceBench}, a benchmark of 75 specialized long-horizon tasks and 154 curated skills across seven domains. We also introduce \textsc{RedAct} https://github.com/XuShuwenn/RedAct, a protected trace release framework that localizes protected key information, rewrites traces while preserving verifier-critical evidence, and embeds behavioral watermarks for downstream provenance analysis. Across representative trace reuse methods, \textsc{RedAct} reduces normalized skill transfer (NST) from 44.7--67.1\% on raw traces to below the no-skill baseline, while preserving audit evidence. Its standalone behavioral watermarks reach 93.6--100.0\% true detection with a false alarm rate of at most 1.9\%. These results frame public agent traces as security interfaces and show that selective redaction can reduce procedural capability leakage without removing audit evidence.

### 1. 论文的核心问题与整体含义 (研究动机和背景)

*   **核心问题**: 智能体执行轨迹富含程序细节（如工具调用、中间决策、错误恢复逻辑），这些细节在提供审计价值的同时，也构成了“秘密攻击面”，可能被下游攻击者利用，以蒸馏、提取和复用其中包含的私有程序技能（如专有公式、校准阈值、工作流等）。这被称为**黑盒轨迹泄露**问题。
*   **研究背景**: 神经符号智能体在专业领域（如生物医学、金融）的应用日益广泛，其有效性高度依赖于编码了专业知识与流程的私有技能包。同时，为了确保透明度、可调试性和用户信任，常常需要发布执行轨迹。这构成了**轨迹可审计性**与**程序技能保护**之间的根本性矛盾。现有研究多聚焦于保护推理文本或模型权重，鲜有关注轨迹本身对程序过程技能的泄露风险。
*   **整体含义**: 本文是首个将智能体轨迹发布系统性地定义为一个资产保护问题的研究。它强调了固有的安全风险，并提供了一个具体的解决方案框架，证明了透明性与安全性可以并行不悖，为构建更安全、负责任的智能体系统提供了重要的理论和实践指导。

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**: 提出 **RedAct** (Redacting Agent Capability Traces) 框架，这是一个受保护的轨迹发布框架。它通过“选择性重写”和“行为水印”两层机制，在发布轨迹时有效地移除或混淆可被复用的程序细节，同时保留审计所需的关键证据。
*   **关键技术细节**:
    1.  **关键信息引导的重写层 (Key-Information-Guided Rewriting)**:
        *   **步骤一：定位 (Key-Item Locator)**。假设技能拥有者可以访问完整的技能包。首先，利用一个基于LLM的“关键信息定位器”（\( g_{\eta} \)），从任务指令和技能文件（\( d, s \)）中识别出受保护的关键程序项 \( K_{prot} \)，如公式、常数、阈值、专用工具选择、验证例程等。随后进行人工审核以去重。
        *   **步骤二：重写 (Rewriter)**。然后，基于 \( K_{prot} \) 和原始轨迹 \( \tau \)，使用一个LLM重写器（\( \rho_{\theta} \)）生成受保护的轨迹 \( z_{RA} \)。重写策略是：**抽象化**暴露受保护信息的中继轮次，将其替换为高级任务描述；**保留**最终答案、工具使用证据和保障审计的顺序。
        *   算法流程：输入任务指令(d)、技能包(S)、轨迹(τ)，输出受保护轨迹(z)。
            *   提取所有技能中的关键项K。
            *   分离轨迹中的用户回合(U)和助手回合(A)。
            *   对每个助手回合批次B，使用重写器(ρθ)结合K进行重写，生成C。
            *   确保C格式正确，与B对齐，然后拼接。
            *   最终验证z与原始τ的最终答案是否一致，确保审计信息完整。
    2.  **行为水印层 (Behavioral Watermarking)**:
        *   **目标**: 追踪下游对受保护轨迹的复用行为，提供溯源证据。
        *   **水印注入**: 定义了几种“钩子家族”（hook families），即功能中立的行为模式。RedAct 选择一个受保护的轨迹子集，在这些轨迹的特定位置（如任务开始或工具调用之后）注入无害的行为模式（如检查环境、添加仪式性标记、确认结果）和对应的激活短语（\( \kappa_h \)）。水印注入算法确保不改变轨迹的最终答案和验证结果。
        *   **水印检测**: 当需要检查一个可疑的学生模型时，使用激活短语（\( \kappa_h \)）提示它生成轨迹，然后应用检测器（\( W_{detect}^h \)）来检查轨迹中是否包含预期水印模式。
        *   **钩子家族类型**: 包括两类四种：
            *   **独立钩子 (Standalone)**：与上下文无关，如 `Ritual Marker`（在任务开始/结束处添加固定标记），`Env Check`（增加无害的环境探测行为）。
            *   **上下文钩子 (Contextual)**：依赖特定上下文触发，如 `Cross Check`（在工具调用后添加验证性语言），`Error Anchoring`（在错误反馈后添加修正性短语）。

### 6. 论文的主要结论与发现

1.  **原始轨迹是脆弱的攻击面**: 原始轨迹携带大量可被复用的程序知识，其性能增益几乎可以媲美直接使用完整技能包（Oracle Skills）。NST高达44.7%-67.1%，说明泄露风险极其严重。
2.  **RedAct有效抑制了复用优势**: RedAct的保护能显著降低NST至**负数**（低于无技能基线），完全消除了原始轨迹带来的复用性能优势。RPI也降低了37-48%，说明关键信息泄露得到有效控制。
3.  **困难任务风险最高但保护效果显著**: 在依赖领域知识的困难任务上，原始轨迹的泄露风险最严重，但RedAct的保护效果也更显著，有时甚至使复用方法性能低于无技能基线。
4.  **行为水印提供有效溯源证据**: 独立水印（如Env Check）达到93.6%-100%检测率、低于1.9%的极低虚警率，提供了强有力的复用证据。上下文水印（如Error Anchoring）虽然检测率较低，但误报率为0%，适用场景不同。
5.  **RedAct保持了审计完整性**: 保护后的轨迹保留了91-96.6%的审计证据（最终答案、工具名称等），同时去除了70.6%的关键敏感内容，表明其实现了可审计性与安全性的良好平衡。

### 8. 不足与局限

*   **评估环境受限**：所有实验均在受控、自动验证器驱动的模拟环境中进行，与真实世界的复杂、嘈杂、动态的实际部署环境存在差距。验证器反馈的强弱和准确性也可能影响结果泛化。
