## 摘要
LLM智能体日益依赖外部推理条件：提示词、工具、记忆、标准操作流程（SOP）、技能和框架反馈。这些资产可以在不改变模型权重的情况下改进任务执行，但通常通过启发式反思或复用观察到的成功与失败（仿佛计数本身就是可靠信念）来修改。我们提出了\textbf{贝叶斯智能体（Bayesian-Agent）}，一个原生且跨框架的框架，将可复用技能和SOP视为关于冻结模型在特定提示词、上下文和框架环境下是否会成功的假设。贝叶斯智能体记录经过验证的轨迹证据，维护每个技能上基于特征的条件类别后验，并将后验状态映射为可检查的动作，如修补、拆分、压缩、淘汰和探索。面向模型的提示词获得可执行的防护栏和失败模式修补，而后验摘要仍可用于审计。使用\texttt{deepseek-v4-flash}，增量修复将SOP-Bench从80\%提升至95\%，Lifelong AgentBench从90\%提升至100\%，RealFin-Bench从45\%提升至65\%。我们进一步评估了贝叶斯智能体的原生后端以及可选的GenericAgent、mini-swe-agent和Claude Code后端。结果包括正面、负面、饱和及案例研究设置，表明智能体技能演化最好被视为后验引导的框架优化，而非未校准的提示词累积。源代码可在https://github.com/DataArcTech/Bayesian-Agent获取。

## Abstract
LLM agents increasingly rely on external inference conditions: prompts, tools, memory, SOPs, skills, and harness feedback. These assets can improve task execution without changing model weights, but they are often revised by heuristic reflection or by reusing observed successes and failures as if counts alone were reliable belief. We introduce \textbf{Bayesian-Agent}, a native and cross-harness framework that treats reusable skills and SOPs as hypotheses about whether a frozen model will succeed under a particular prompt, context, and harness environment. Bayesian-Agent records verified trajectory evidence, maintains a feature-conditioned categorical posterior over each skill, and maps posterior state into inspectable actions such as patch, split, compress, retire, and explore. Model-facing prompts receive executable guardrails and failure-mode patches, while posterior summaries remain available for audit. With \texttt{deepseek-v4-flash}, incremental repair improves SOP-Bench from 80\% to 95\%, Lifelong AgentBench from 90\% to 100\%, and RealFin-Bench from 45\% to 65\%. We further evaluate Bayesian-Agent's native backend and optional GenericAgent, mini-swe-agent, and Claude Code backends. The results include positive, negative, saturated, and case-study settings, suggesting that agent skill evolution is best viewed as posterior-guided harness optimization rather than uncalibrated prompt accumulation. The source code is available at https://github.com/DataArcTech/Bayesian-Agent.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **动机**：现代 LLM 智能体（Agent）的性能日益依赖于外部推理条件（提示词、工具、记忆、SOP、技能、框架反馈等），而不只是模型权重。这些外部资产在任务执行中起关键作用，但现有的维护方式存在严重缺陷：要么通过启发式自我反思进行修改，要么简单复用观察到的成功/失败计数（认为计数自身就是可靠信念）。这种未校准的累积方式会导致噪声编辑、过时假设、性能不稳定等问题。
- **核心问题**：如何让智能体框架在保持模型权重冻结的前提下，基于可验证的证据（verified trajectories）来科学、可审计地演化其外部技能和 SOP，使技能更新成为后验引导的优化过程，而非简单的文本堆砌。
- **整体含义**：提出将技能视为“承载证据的假设”（evidence-bearing hypotheses），通过贝叶斯后验更新来量化不确定性，并以此驱动具体的维护动作（修补、拆分、压缩、淘汰、探索），从而实现可审计、可解释的技能演化。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- **形式化视角**：将可复用技能 \(h_k\) 视为关于“冻结模型在给定提示、上下文、框架环境下是否会成功”的假设。对每个技能维护一个基于特征（feature-conditioned）的类别后验分布，后验状态映射为具体的可检查动作。
- **核心理念**：智能体演化本质上是“后验引导的框架优化”（posterior-guided harness optimization），而非未校准的提示累积。框架维护一个轨迹证据库，通过贝叶斯证据模型更新对技能可靠性的信念，然后由策略决定执行何种修改。

### 2.2 关键技术细节

1. **轨迹证据模型**：每个轨迹记录为 \(e_t = (x_t, h_k, c_t, y_t, u_t, \tau_t, \ell_t, r_t, m_t)\)，其中 \(y_t\) 是外部验证器提供的二进制结果（非模型自评），\(r_t\) 是失败模式。特征映射 \(g(e_t)\) 将运行时信号（上下文、失败模式、token 数量桶、轮次数桶、延迟桶、短标量元数据）离散化。

2. **贝叶斯证据模型**：
   - 采用特征条件类别模型，使用拉普拉斯平滑（\(\lambda=1\)）计算类别先验 \(\pi_{k,t}(\ell)\) 和条件似然 \(\theta^{(l)}_{k,j,t}(v)\)。
   - 后验分值：\(\tilde{p}_{k,t}(\ell|z) = \pi_{k,t}(\ell) \prod_{j=1}^m \theta^{(l)}_{k,j,t}(z_j)\)，归一化后得到成功后验 \(s_{k,t}(z)\)。
   - 同时维护共轭 Beta-Bernoulli 摘要（\(\alpha_{k,t}, \beta_{k,t}\)）用于保守的失败优势检查。

3. **后验引导的技能动作**：决策策略为有序规则：
   - **探索（Explore）**：无观测时触发。
   - **淘汰（Retire）**：\(\beta \ge 4\) 且成功概率 < 0.45 时触发。
   - **修补（Patch）**：同一失败模式出现至少 2 次时触发，将具体失败模式转化为可执行防护栏。
   - **拆分（Split）**：至少 3 个上下文和 4 个观测表明技能覆盖异质情况时触发。
   - **压缩（Compress）**：至少 3 个观测且成功概率 \(\ge 0.72\) 时触发，保持技能简洁。

4. **模型-框架边界**：后验审计信息（成功率、成本、失败模式等）存储于注册表用于审计；但 LLM 提示词仅接收可执行技能文本（稳定防护栏 + 失败模式修补），避免让模型直接推理后验数值。

5. **执行模式**：
   - **全量模式（Full）**：注册表从空开始在线更新。
   - **增量模式（Incremental）**：先读取现有智能体运行轨迹，更新注册表，然后仅重跑失败任务（使用后验引导的修补上下文）。

### 2.3 算法流程概述

算法 1（Bayesian-Agent 演化）：
- 初始化空注册表 \(B\)。
- 若增量模式，则从基线轨迹中验证所有样本，更新计数；待重跑任务集 = 失败任务。
- 遍历待执行任务：选择相关技能和其后验状态；按策略选择动作；渲染面向模型的技能上下文（防护栏+修补）；保存前后快照；执行任务并验证；更新证据库。
