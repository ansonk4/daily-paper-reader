## 摘要
使用工具的大型语言模型（LLM）智能体面临两种不同的安全缺陷：未经授权的外部操作，以及在最终输出检查介入之前运行时内部敏感明文暴露。现有防御通常只保护一个边界（规划器/运行时或动作端点），因此本身无法同时保护这两个层面。我们提出SecureClaw，一种双边界架构，在效果端点实施授权，在读取边界实施明文隔离。敏感读取通过一个可信网关，该网关用不透明句柄替换原始值，并在评估部署中，将有限摘要作为明确的降级接口。改变外部状态的写入遵循PREVIEW→COMMIT协议，其中只有可信执行器才能提交经策略授权的确切规范请求。运行时仍可基于摘要和符号引用进行规划，但无法直接解引用秘密或执行副作用。在AgentDojo、AgentLeak和Agent Security Bench (ASB)上，SecureClaw是我们在公共测试框架中评估的唯一一种防御，它能同时保留可用的任务效用，并在ASB上实现0%的攻击成功率（ASR），在AgentDojo上实现0.64%的ASR，在AgentLeak的攻击并行测试通道上实现3.23%的整体泄露（衡量最终输出和内部中继泄露）。

## Abstract
Tool-using large language model (LLM) agents face two distinct security failures: unauthorized external actions and exposure of sensitive plaintext inside the runtime before any final output check can intervene. Existing defenses usually protect one boundary, either the planner/runtime or the action sink, and therefore do not by themselves secure both surfaces. We present SecureClaw, a dual-boundary architecture that places authorization at the effect sink and plaintext confinement at the read boundary. Sensitive reads pass through a trusted gateway that replaces raw values with opaque handles and, in the evaluated deployment, bounded summaries as an explicit declassification interface. Writes that change external state follow a PREVIEW$\rightarrow$COMMIT protocol in which only a trusted executor may commit the exact canonical request authorized by policy. The runtime can still plan over summaries and symbolic references, but cannot directly dereference secrets or perform side effects. Across AgentDojo, AgentLeak, and Agent Security Bench (ASB), SecureClaw is the only defense we evaluate in a common harness that simultaneously retains usable task utility and achieves 0\% attack success rate (ASR) on ASB, 0.64\% ASR on AgentDojo, and 3.23\% overall leak on AgentLeak's attacked parity lane, which measures final-output and internal-relay leakage.

### 论文的核心问题与整体含义（研究动机和背景）

-   **核心问题**：当前使用工具的大型语言模型（LLM）智能体面临两种关键的、且相互独立的安全威胁：
    1.  **未授权的外部动作（Effect Surface）**：通过提示注入（Prompt Injection）等手段，攻击者可以操纵智能体向外部系统（如发送邮件、修改文件）执行未经授权的操作，造成不可逆的后果。
    2.  **运行时敏感明文泄露（Plaintext Surface）**：在最终输出被检查之前，智能体运行时环境（Runtime）内部的敏感数据（如邮件正文、发票详情）可能通过内部通道（如智能体间消息C2、共享内存C5）泄露给攻击者。这种泄露在输出检查环节无法被阻断。
-   **现有方法不足**：现有的防御措施通常只保护一个边界（要么保护效果端点，要么保护运行时本身），无法同时应对上述两种威胁。例如，仅对输出进行过滤无法阻止内部泄露；仅对动作进行授权，无法阻止运行时的敏感数据被读取和中继。
-   **论文目标**：提出一种**双边界（Dual-Boundary）架构**，在效果端点和读取端点分别实施控制，同时解决未授权执行和运行时明文泄露这两个根本性问题，从而“夺回”对LLM智能体的控制权。

### 论文提出的方法论：核心思想、关键技术细节

1.  **核心思想**：将“执行外部动作的能力”与“访问内部敏感数据的能力”彻底分离给不同的可信组件，确保不受信任的智能体运行时（Runtime）无法同时拥有这两项权力。

2.  **关键技术细节**：
    -   **架构组成**：系统由**未受信任的运行时（Runtime）**、**可信网关（Gateway）**、**可信句柄存储（Handle Store）**、**策略引擎（Policy Engine）** 和**可信执行器（Executor）** 五部分构成。
    -   **读边界（Read Boundary）保护 - 防止明文泄露**：
        -   **句柄化**：当智能体读取敏感数据时，可信网关拦截请求。它将原始敏感数据存储在自己的存储中，并为之创建一个**不透明句柄（Opaque Handle）**（一个高熵的符号引用，如`hid = HMAC-SHA256(k_handle, nonce || session || object)`）。运行时只能看到这个句柄，而无法看到原始明文。
        -   **显式降级（Explicit Declassification）**：为了让智能体仍能完成需要理解内容的规划任务，网关会选择性地返回一个**有限摘要**（Bounded Summary）。这个摘要由固定的、受策略约束的规则生成（如：“来自供应商A的发票，金额$4.2k，周五到期”），被视为一个显式的、可审计的降级通道，而非原始文本的意外泄露。
    -   **写边界（Write Path）保护 - 防止未授权动作**：
        -   **PREVIEW → COMMIT 协议**：
            1.  **预览（PREVIEW）**：运行时可以提议一个动作，但必须**通过网关**提交包含`(意图, 调用者, 会话, 输入, 上下文)`的完整请求（ρ）。
            2.  **规范化与绑定**：网关将请求规范化为标准格式（Canon(ρ)），计算其绑定摘要 `h = HMAC-SHA256(k_bind, Canon(ρ))`。策略引擎对规范化的请求进行授权，并返回一个绑定此摘要 `h` 的授权工件（Authorization Artifact）。
            3.  **提交（COMMIT）**：可信执行器是**唯一能执行外部动作**的组件。执行器收到动作提议后，会**重新计算**请求的绑定摘要 `h`，并验证授权工件。只有当授权有效、请求未被篡改、时间戳新鲜、并满足重放保护等所有条件时，才会实际执行该外部效果。
            4.  **拒绝后恢复（Deny-aware Recovery）**：如果执行器拒绝了一个提交，它不会简单地终止任务，而是返回一个结构化的拒绝信息（如动作类别、拒绝原因），引导智能体走向安全的替代方案（如“改为起草待确认”），这在基准测试中显著恢复了被阻塞攻击后的任务效用。

### 论文的主要结论与发现

1.  **核心结论**：工具型LLM智能体需要同时保护“效果表面”和“明文表面”，这两个表面不可或缺，且不能互相替代。**SecureClaw是唯一一个在统一框架下同时防御这两种安全失败的架构**。
2.  **主要实验结果**：
    -   在**ASB**上实现**0%** 的攻击成功率（ASR），同时保持**88.90%** 的任务效用。

### 优点

-   **系统性强，直指根本问题**：精准识别了agent安全的两个独立且关键的失效模式，并针对性地设计了双边界架构，而不是头痛医头脚痛医脚。

### 不足与局限

1.  **良性任务性能下降**：虽然安全性能极佳，但在AgentDojo上的良性任务效用（60.82%）低于无防御的Plain模型（70.10%），说明句柄化和摘要化过程对某些任务带来了开销，简单任务的开销尤其明显。
2.  **降级摘要的信息泄露**：尽管摘要被认为是显式降级，但论文自己的测量显示，在部分场景下（如自由文本摘要），摘要仍会区分秘密信息（Δ=1），这是其最直接的泄露途径。严格的安全应用需要更严格的摘要策略。
3.  **信任假设**：SecureClaw将信任集中于网关、策略引擎和执行器。如果这些组件中的任何一个被攻破，其安全保证将全部失效。文中附录B.6也详细分析了这一点。
4.  **策略的粗粒度**：失败的案例（如向错误的人发送了允许的邮件）表明，其安全上限受限于策略引擎的粒度。需要更细致的策略绑定（如对象级别、阶段级别）才能进一步提升。
5.  **单模型评估为主**：主要基准测试全部基于`gpt-4o-mini`，虽然做了小规模跨模型验证，但在其他模型族上的泛化性能评估不够充分。
6.  **成本与部署**：引入双边界架构增加了系统的复杂性和少量延迟（虽然文中论证了延迟可接受），在实际高并发部署中的工程挑战和成本未被讨论。
