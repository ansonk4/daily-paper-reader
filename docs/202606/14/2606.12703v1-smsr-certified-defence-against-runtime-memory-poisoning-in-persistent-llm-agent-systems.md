## 摘要
检索增强生成（RAG）智能体越来越多地运行于跨用户会话积累的持久化内存上。这创造了一个新的攻击面：仅通过正常渠道交互的对手可以注入精心构造的记忆，一旦被检索，这些记忆就会引导智能体对未来用户的响应，而无需接触模型权重或代码。我们称此为多会话内存投毒（MSMP），并表明现有防御无法对其进行认证；静态语料防御（RobustRAG、ReliabilityRAG）假设固定知识库，而启发式过滤器则被流利的商务风格文本绕过。我们提出了带签名的记忆与平滑检索（SMSR），这是首个针对该场景具有认证鲁棒性边界的防御。组件1在写入时添加HMAC-SHA256来源证明，阻止未签名注入。组件2在查询时应用基于判决多数投票的随机记忆消融，限制认证对手的影响。我们证明无来源证明的检索时过滤器无法认证对抗自适应注入，推导了组件2的超几何认证，并形式化了一致少数效应，即一致对抗答案作为数值少数会赢得基于字符串的投票，而基于判决的投票则消除它。在15个企业场景（3,150次重复试验）中，组件1将未签名变体的攻击成功率从93-100%降至0%。对于单次注入的认证对手，组件2将成功率控制在8.0%（95% CI [5.8, 10.9], n=450），低于认证最坏情况。在端到端仅查询攻击中（智能体自身编写毒物而非预植入），SMSR在实时智能体堆栈上将成功率从65.3%降至5.3%（n=150，非重叠置信区间）。干净查询效用为90%（组件1）和85%（组合）。

## Abstract
Retrieval-augmented generation (RAG) agents increasingly run with persistent memory that accumulates across user sessions. This creates a new attack surface: an adversary interacting only through normal channels can inject crafted memories that, once retrieved, steer the agent's responses for future users, without touching model weights or code. We call this Multi-Session Memory Poisoning (MSMP) and show that no existing defence certifies against it; static-corpus defences (RobustRAG, ReliabilityRAG) assume a fixed knowledge base, and heuristic filters are bypassed by fluent enterprise-style text. We present Signed Memory with Smoothed Retrieval (SMSR), the first defence with a certified robustness bound for this setting. Component 1 adds HMAC-SHA256 provenance at write time, blocking unsigned injection. Component 2 applies randomised memory ablation with verdict-based majority voting at query time, bounding the influence of authenticated adversaries. We prove that no provenance-free retrieval-time filter can certify against adaptive injection, derive a hypergeometric certificate for Component 2, and formalise the Consistent Minority Effect, whereby a consistent adversarial answer wins string-based voting as a numerical minority while verdict-based voting removes it. Across 15 enterprise scenarios (3,150 repeated trials), Component 1 cuts attack success from 93-100% to 0% for all unsigned variants. For an authenticated adversary with a single injection, Component 2 holds success to 8.0% (95% CI [5.8, 10.9], n=450), below the certified worst case. In an end-to-end query-only attack where the agent itself writes the poison rather than it being pre-seeded, SMSR reduces success from 65.3% to 5.3% (n=150, non-overlapping CIs) on a live agent stack. Clean-query utility is 90% (Component 1) and 85% (combined).

## 一、论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：检索增强生成（RAG）智能体系统广泛部署于企业环境，其持久化记忆存储会跨用户会话积累。攻击者可以通过正常交互渠道注入精心构造的“记忆”条目，当未来用户的查询匹配该条目时，智能体会输出恶意响应（如虚假政策、错误限额），且攻击者无需修改模型权重或代码。这种 **多会话内存投毒（MSMP）** 威胁缺乏有效的认证防御。
- **现有不足**：RobustRAG、ReliabilityRAG 等静态语料防御假设知识库固定不变，无法应对运行时动态注入；启发式过滤器（关键词黑名单、困惑度过滤、语义异常检测）能被流利的商务风格文本轻松绕过；A-MemGuard 等共识验证方法仅在经验上表现良好，但缺乏正式安全保证。
- **研究意义**：首次为 MSMP 场景提供**可认证的鲁棒性边界**（certified robustness bound），填补了动态内存投毒认证防御的空白，可部署为 RAG 记忆存储的即插即用 wrapper，无需修改底层 LLM 或嵌入模型。

## 二、论文提出的方法论

### 核心思想
SMSR 由两个组件构成，分别在**写入时**和**查询时**提供防御：

- **组件 1（HMAC 来源标记）**：在写入时，每个合法记忆条目都通过服务器端的 HMAC-SHA256 签名接收标签，只有带有效签名的条目才能在检索时被纳入候选池。从而彻底阻断未签名的注入攻击。
- **组件 2（随机化内存消融 + 基于判决的多数投票）**：对已认证的对手（拥有合法 HMAC 签名但仍可注入恶意条目），采用**过取随机化**策略：检索时先取回 m 个候选条目（m > k），然后在每一轮从这 m 个条目中**均匀无放回**地采样 k 个，多次运行后获得多个响应。使用 LLM 作为判决器对每个响应给出“正确/恶意/无法判断”标签，最后按判决结果的多数决定最终输出。

### 关键技术细节
- **HMAC 标签结构**：τ = HMAC_K(content ∥ session_id ∥ timestamp)，使用 256 位密钥 K。
- **组件 2 参数**：m 为过取池大小，k 为每轮上下文大小，n_runs 为独立运行次数。核心公式：p_clean = C(m - t', k) / C(m, k)，其中 t' 为候选池中恶意条目数。超几何分布推导出最坏情况下的恶意输出概率 δ。
- **一致少数效应（CME）**：论文形式化描述了当恶意输出在文本上高度一致，而良性输出在表述上高度多样时，基于字符串的多数投票会被少量一致恶意输出“劫持”。而基于判决的投票按标签聚合，消除了该效应。

## 七、优点

- **首个可认证防御**：为动态内存投毒场景提供了形式化安全边界，而不仅仅是经验性防御。
- **双组件设计清晰**：组件 1 解决未签名注入（必要且完备），组件 2 解决认证注入（有界对手）。
- **理论证明严谨**：给出了无来源证明过滤器的不可能性定理（定理 1）和组件 2 的超几何证书（定理 2）。
- **一致少数效应的形式化与消解**：深入分析了一个实际投票漏洞，并基于判决投票消除了它。
- **实验覆盖广泛**：3,150 次重复试验，多种配置和攻击类，提供置信区间。
- **端到端验证**：模拟真实攻击流程（查询注入而非预植入），显著提升外部效度。
- **低部署成本**：无需修改模型，即插即用；计算开销在企业场景下可接受。

## 八、不足与局限

- **证书仅对小 t 有意义**：当恶意条目数达到 m/2 时 δ≈1，防御失效。实际场景依赖写入配额、去重、审计日志等企业控制来限制 t。
- **模拟企业数据**：非真实企业部署，场景简洁，可能不能完全反映真实语境下的判决复杂性和响应多样性。
- **启发式基线过于简单**：对比的启发式防御仅包括基础黑名单/困惑度/语义异常，未包含更先进的异常检测方法。
- **未收集人类标注**：依赖 LLM 判决器，虽经 κ 验证，但判决器自身可能被适应对手欺骗（论文承认这一点）。
- **存储开销**：HMAC 标签和去重日志增加了存储和索引开销，文中未量化。
- **密钥管理风险**：HMAC 密钥泄漏导致组件 1 完全失效；密钥轮换会无效所有历史记忆，影响可用性（论文提出 30-90 天轮换，但未评估影响）。
- **未覆盖训练数据投毒**：SMSR 只保护检索层，无法防御 LLM 参数知识被投毒。
- **自适应对手考虑不足**：仅提到旋转判决模型和随机化系统提示可增加难度，未做对抗性响应构造实验。
