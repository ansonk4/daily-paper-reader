## 摘要
间接提示注入攻击通过在智能体任务执行过程中检索的第三方数据中嵌入恶意指令，劫持基于LLM的智能体。现有防御在静态基准上报告了接近零的攻击成功率，但最近的自适应评估显示，一旦允许攻击者针对部署的防御进行优化，这些结果就会崩溃。在这项工作中，我们将这种崩溃归因于两种失败模式。首先，现有防御方法局限于识别特定的攻击模式，而不是评估每个嵌入指令的意图是否与用户任务相关。其次，基于训练的防御方法（尽管在安全-效用权衡方面最强）从少量手工制作的模板中组装对抗性样本，导致防御者无法推广到该狭窄策略分布之外。为了解决这些差距，我们提出了RETA，一种基于训练的方法，将防御决策建立在用户任务而非攻击者控制的数据上。在每个工具输出步骤，防御者进行链式推理，验证其行为是否与用户任务一致。利用红队测试，模拟攻击者合成对抗训练数据，并接收字典学习多样性奖励，实现了注入重表述策略的广泛覆盖。这些共同使得防御者可以通过多目标强化学习进行优化，实现更好的安全-效用权衡。在六种黑盒自适应攻击中，RETA使每个攻击的ASR保持在10%以下，在两个目标模型上的平均ASR分别为2.92%和3.75%，同时在攻击下和干净输入上保持了大部分效用。

## Abstract
Indirect prompt injection attacks hijack LLM-based agents by embedding malicious instructions in third-party data that the agent retrieves during task execution. Existing defenses report near-zero attack success rate on static benchmarks, yet recent adaptive evaluations show that these results collapse once the attacker is allowed to optimize against the deployed defense. In this work, we trace this collapse to two failure modes. First, existing defense methods are confined to recognizing specific attack patterns, rather than assessing whether the intent of every embedded instruction is relevant to the user task. Second, training-based defenses, which otherwise offer the strongest safety-utility trade-off, assemble their adversarial examples from a handful of hand-crafted templates, and the resulting defender fails to generalize outside that narrow strategy distribution. To address these gaps, we propose RETA, a training-based method that grounds defense decisions on the user tasks rather than attacker-controlled data. At each tool-output step, the defender undertakes chain-of-thought reasoning verifying that its actions are consistent with the user task. Leveraging red-teaming, a simulated attacker synthesizes adversarial training data and receives a dictionary-learning diversity reward, achieving broad coverage of injection-reformulation strategies. Together, these allow the defender to be optimized via multi-objective reinforcement learning and achieve better safety-utility trade-off. Across six black-box adaptive attacks, RETA keeps every per-attack ASR below 10%, with average ASR of 2.92% and 3.75% on the two target models, while preserving most utility under attack and on clean inputs.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究问题**：间接提示注入攻击（Indirect Prompt Injection, PI）通过将恶意指令嵌入智能体任务执行时检索的第三方数据（如网页、API响应、数据库记录），劫持基于LLM的智能体。现有防御措施在静态基准上报告了接近零的攻击成功率（ASR），但一旦允许攻击者针对部署的防御进行自适应优化，这些结果迅速崩溃。
- **识别出的两大失败模式**：
  1. 现有防御仅识别攻击的表面模式（如特定措辞），而非评估每个嵌入指令的意图是否与用户任务相关，因此攻击者可通过重写注入的表述轻易绕过。
  2. 基于训练的防御方法（如SecAlign等）虽然安全-效用权衡最强，但其对抗训练数据仅来自少量手工模板或单一优化成功的攻击策略，导致决策边界只覆盖狭窄的策略分布，无法泛化到未见过的攻击策略。
- **整体含义**：静态基准高估了防御鲁棒性；必须从任务对齐的角度重新设计防御，使其通过推理验证动作是否源自用户任务，而非仅依赖攻击者控制的数据表面。

## 2. 方法论

### 2.1 核心思想
RETA（Reasoning-enabled Task Alignment）通过两阶段对抗性强化学习训练防御者：

- **阶段 I（红队对抗样本生成）**：使用一个可学习的攻击者策略，在冻结的基线模型上生成成功注入，通过字典学习多样性奖励确保覆盖不同的注入重表述策略，避免模式坍塌。
- **阶段 II（防御者强化学习）**：防御者基于阶段I收集的对抗样本进行训练，在完整轨迹上同时优化三个奖励：拒绝注入任务（安全）、完成用户任务（效用）、在决策前输出任务对齐的链式推理（验证下一步工具调用是否可从用户任务推导出）。

### 2.2 关键技术细节

- **任务对齐推理**：防御者在每次动作前输出 `<reason>...</reason>` 形式的推理，包含两个条件：
  - C1：明确识别当前观测中试图重定向的内容。
  - C2：论证下一步工具调用集仅由用户任务决定，不受指令性内容影响。
- **字典学习多样性奖励**：从成功攻击的策略记录中学习一套可解释的字典基元，对每个新生成的注入，用稀疏编码评估其策略被当前字典解释的程度，未被很好解释的注入获得更高多样性奖励，从而驱动攻击者探索新策略。
- **多目标强化学习**：使用 GDPO（Group reward-Decoupled Normalization Policy Optimization），对每个奖励独立进行组归一化后再加权，安全与效用奖励权重最高（1.0），推理奖励权重次之（0.5），格式奖励最低（0.2）。
- **整体流程**（Algorithm 1）：
  1. 攻击者 SFT 热启动；
  2. 每 50 步刷新字典从累积的未解释成功策略记录中学习；
  3. 阶段 I 运行 250 步，每步用 GDPO 更新攻击者，并收集所有成功的 `(context, injection)` 对到 `D_RT`；
  4. 防御者 SFT 热启动（基于边界案例的重写），然后在 `D_RT` 上运行 250 步 GDPO。

## 6. 主要结论与发现

- **RETA 在所有评估的攻击上**（六种自适应攻击、五种静态攻击、两个跨基准）均达到最低 ASR，且每个攻击的 ASR 低于 10%。
  - Qwen3-4B：平均自适应 ASR 2.92%（最高 7.50%），静态 ASR 2.90%。
  - Llama-3.1-8B：平均自适应 ASR 3.75%（最高 5.00%），静态 ASR 1.76%。
- 效用方面：在攻击下 UA 保持接近无防御基线（Qwen: 55.8% vs 57.5%；Llama: 49.2% vs 47.1%），干净输入 BU 仅小幅下降（Qwen: 60.8% vs 61.9%；Llama: 45.4% vs 48.5%）。
- 成功自适应攻击所需的中位迭代次数显著高于其他防御（Qwen: 19 vs MetaSecAlign 的 11）。
- 关键发现：**任务对齐推理和策略覆盖多样性是获得泛化鲁棒性的两个必要组件**；仅增加训练数据量无法替代策略覆盖。

## 7. 优点

1. **创新性的任务对齐推理**：首次将 PI 防御从模式识别提升到基于用户任务的推理，稳定参考点（用户任务）无法被攻击者改写。
2. **字典学习多样性奖励**：解决了强化学习攻击者模式坍塌的问题，驱动攻击者探索未覆盖的重表述策略，使训练分布更广。
3. **多目标强化学习框架（GDPO）**：直接优化安全、效用、推理三个目标，实现优于单一目标或 SFT 的权衡。
4. **两阶段训练流程清晰**：阶段 I 红队收集多样对抗样本，阶段 II 防御者在此基础上学习，无需复杂共训练。
5. **实验设计完整**：覆盖多种自适应攻击、静态攻击、跨基准迁移、消融研究，结果详实。
6. **开源**：提供了匿名化工件，促进可复现性。

## 8. 不足与局限

1. **自适应评估子集规模小**：自适应攻击仅运行在 40 例子集上（虽然刻意选择脆弱案例），未来应扩展至全套件。
2. **推理-动作不匹配**：部分失败案例中模型能识别注入但仍执行攻击动作，表明当前轨迹级奖励未能约束中间步骤行为，需设计步级奖励。
3. **过度防御导致的效用损失**：模型有时将外来任务参数误判为不安全，拒绝后也放弃用户任务，导致 UA 和 BU 轻微下降。需要更细粒度的任务进度奖励。
4. **单轮会话限制**：仅评估单次用户-智能体会话，未覆盖多轮、持久性攻击（如记忆状态、跨会话影响），实际部署中此类攻击更常见。
5. **与系统级防御比较缺失**：未与 CaMeL、IPIGuard 等改变执行模型的方法比较，因为当前基准难以提供公平比较场景（数据依赖的安全-效用任务）。
6. **推理成本开销**：任务对齐推理增加了延迟和 token 使用（表10），在安全敏感的 workflow 上可接受，但在延迟敏感场景需优化（如使用条件推理、长度惩罚）。
7. **依赖辅助 LLM**：字典学习、策略提取、推理判断等使用额外 LLM（Qwen3-1.7B 和 14B），增加了系统复杂性和潜在级联失败点。
8. **未探索更大模型**：仅评估 4B 和 8B 规模，更大模型（70B+）的行为可能不同，需进一步验证。
