## 摘要
大语言模型在多轮交互中维护不断演化的上下文而非生成孤立响应，这使得它们容易受到提示注入和上下文投毒攻击——局部看似合理的对抗性片段逐渐扭曲推理轨迹。现有防御主要过滤单个输出，往往忽略跨轮次的上下文演化，使长程推理暴露在风险中。尽管模型上下文协议标准化了上下文交换和工具调用，但它仅作为被动路由层，并未强制上下文稳定性。为解决这些局限，我们提出博弈论安全模型上下文协议，一种控制器驱动的多智能体方法，将上下文管理视为闭环动态过程。GT-MCP协调三个异质LLM智能体，并通过信任函数选择输出，该函数联合评估针对验证上下文图表的因果一致性、智能体间的语义一致性以及随时间推移的分布漂移。当检测到不稳定时，基于回滚的自我修复机制恢复已验证上下文，阻止无支持片段传播。在自适应对抗威胁模型下，对500轮交互的实证评估显示：99.6%的轮次中上下文漂移保持有界，仅0.4%需要恢复。每轮效用保持高度集中，中位数=-0.19，P05=-0.72，P95=0.30；仅0.4%案例出现低于-1的严重退化，且无注入尝试在控制器层面成功。所选输出保持稳定胜率高于98%，计算开销可预测，每令牌延迟=1.63e-3秒。

## Abstract
Large Language Models (LLMs) in multi-turn interactions maintain evolving context rather than generating isolated responses, making them vulnerable to prompt-injection and context-poisoning attacks in which locally plausible adversarial fragments gradually distort reasoning trajectories. Existing defenses mainly filter individual outputs and often ignore context evolution across turns, leaving long-horizon reasoning exposed. Although the Model Context Protocol (MCP) standardizes context exchange and tool invocation, it functions as a passive routing layer and does not enforce contextual stability. To address these limitations, we introduce the Game-Theoretic Secure Model Context Protocol (GT-MCP), a controller-driven multi-agent method that treats context management as a closed-loop dynamical process. GT-MCP coordinates three heterogeneous LLM agents and selects outputs through a trust function that jointly evaluates causal consistency against a validated context graph, semantic agreement among agents, and distributional drift over time. When instability is detected, a rollback-based self-healing mechanism restores the validated context and prevents unsupported fragments from propagating. Empirical evaluation over 500 interaction turns under an adaptive adversarial threat model shows that contextual drift remains bounded in 99.6% of turns, with recovery required in only 0.4%. Per-turn utility remains tightly concentrated, with median = -0.19, P05 = -0.72, and P95 = 0.30; severe degradation below -1 occurs in only 0.4% of cases, and no injection attempt succeeds at the controller level. Selected outputs maintain stable win rates above 98%, and computational overhead remains predictable, with latency per token = 1.63e-3 s.

## 二、方法论：核心思想、关键技术细节与算法流程

### 核心思想
- 将LLM多轮推理视为一个**受扰动动态系统的状态控制问题**。系统维护一个“已验证上下文状态”\(c_t\)，每个新观察到的上下文\(\tilde{c}_t = c_t \oplus \Delta c_t\)可能包含不可信流入\(\Delta c_t\)。控制器只允许通过信任评估的候选响应\(\hat{y}_t\)更新持久状态：\(c_{t+1} = \mathcal{U}(c_t, \hat{y}_t)\)。
- 采用**博弈论框架**：将交互建模为重复Stackelberg博弈。控制器承诺一个选择与恢复策略\(\pi\)，攻击者根据历史响应自适应地产生有界扰动。控制器最小化本地输出风险与长程上下文偏离之和。

### 关键技术组件

1. **因果上下文图 \(G_t = (V_t, E_t)\)**  
   - 节点表示原子声明（subject–predicate–object命题），边表示蕴含/因果关系。每个节点有置信权重\(w_v\)，基于父节点支持、持久性、智能体间一致性和矛盾惩罚计算。
   - 声明提取通过映射\(\mathcal{E}(y)\)，并强制声明预算\(|\mathcal{S}(y)| \leq B\)，防止冗长操纵。

2. **多智能体生成**  
   - 三个异质LLM智能体（GPT-5.3、Llama-3.1-70B、DeepSeek-R1）接收相同输入对\((q_t, \tilde{c}_t)\)，各自生成候选响应\(y_i(t)\)。

3. **信任分数**  
   - 对每个候选，先构建试探性上下文更新\(c_{t+1}^{(i)} = \mathcal{U}(c_t, y_i(t))\)，计算候选特定漂移\(\text{CDS}_i(t) = D_{JS}(\tilde{P}_{t+1}^{(i)} \| P_t)\)（Jensen–Shannon散度）。
   - 信任分数：\(T_i(t) = \alpha \text{CCI}_i(t) + \beta \text{AGR}_i(t) - \gamma \text{CDS}_i(t)\)  
     - **CCI**（因果一致性）：候选声明在图中的父节点支持均值。  
     - **AGR**（跨智能体一致性）：候选与其他候选的语义一致性核（蕴含、余弦相似、矛盾惩罚）。  
     - **CDS**（候选特定漂移）：若该候选被接受后与当前验证状态的距离。
   - 控制器选择信任最高的候选：\(\hat{i} = \arg\max_i T_i(t)\)，只有该输出可更新持久上下文。

4. **自愈恢复**  
   - 若所选输出的\(\text{CDS}_{\hat{i}}(t) > \delta_c\)，触发恢复。
   - 识别低支持节点和矛盾节点，隔离相关片段。
   - 选择回滚深度\(d^* = \arg\min_{d\in[0,D]} [\hat{\mathcal{L}}(G_{t-d}) + \lambda_R d]\)，平衡结构脆弱性与信息损失。
   - 恢复检查点后，重新整合仍能通过因果图验证的片段，丢弃不可信部分。

5. **长程时间稳定性目标**  
   - 累积回报\(V_T\)包含所有轮的CCI、AGR、CDS、攻击成功指示和恢复触发，通过折扣因子\(\rho_T\)加权。

### 算法流程（文字说明）
- **每轮开始**：从用户查询\(q_t\)和已验证上下文\(c_t\)出发，接收不可信流入\(\Delta c_t\)，构造观察上下文\(\tilde{c}_t\)（用\(\oplus\)保留来源标记）。
- **更新因果图**\(G_t\)（从\(c_t\)构建）。
- **并行生成**：三个智能体分别产生候选\(y_i(t)\)。
- **提取声明、计算CCI、试探性更新、计算CDS_i**。
- **计算AGR_i**（两两语义一致性平均）。
- **聚合信任**\(T_i(t)\)，选择最高候选\(\hat{y}_t\)。
- **若CDS > δ_c**：执行自愈（回滚+隔离）；否则直接接受。
- **只有接受或愈合后的上下文**\(c_{t+1}\)进入下一轮，并记录审计日志。

## 三、实验设计

- **稳定轮次比例**：无持续性攻击影响、不超过恢复阈值、无未支持片段插入的轮次。

## 五、实验数量与充分性

- **主实验**：1个环境（500轮，10种子），对比8种方法（4基线+4消融+完整），共9×5000=45000轮观测。
- **消融实验**：系统地移除核心组件（No-CCI, No-AGR, No-CDS, No-Heal），验证每个组件的贡献。
- **多维度分析**：还进行了多智能体选择动态、全局效用分布、延迟效率、推理结构（流形分析）、博弈论结果、基于基准攻击风格的子分析、失败案例总结。
- **充分性评价**：实验设计较为充分和公平：使用相同种子配对比较，覆盖多种攻击类型（包括潜伏型和渐进型），指标全面。但仍存在局限：
  - 缺少更长轮次（如1000或2000轮）测试，无法验证极长程漂移累积。
  - 仅使用固定三个模型组合，未测试其他模型或不同数量智能体。
  - 未与最先进的多智能体防御框架（如Task Shield、SecInfer等）进行直接定量比较，仅提供了功能对比表。
  - 未分别在纯良性环境和纯对抗环境下测试，无法完全隔离正常性能与鲁棒性。

---

## 六、主要结论与发现

1. **上下文稳定性**：99.6%轮次保持近零漂移，仅0.4%（2轮）触发高漂移并成功恢复，无持续性记忆污染。
2. **注入成功率**：控制器层面观察到**0%**注入成功（ISR=0）。
3. **效用分布**：高度集中（中位数-0.19，P05=−0.72，P95=0.30），严重退化仅限0.4%轮次。
4. **多智能体选择**：三个智能体均被选择，但选择基于上下文而非固定排序；DeepSeek-R1表现出最低方差和最高稳定胜率。
5. **效率**：延迟可预测（均值1.63e-3 s/token），额外计算主要流向结构性验证而非噪音放大。
6. **博弈论结果**：攻击者效用均值仅0.001，无法获得系统性的长期回报。
7. **消融验证**：所有组件均不可或缺，去掉CDS导致最大性能下降（ISR提高到4.5%，平均CDS=2.07）。

## 七、优点

- **创新性问题抽象**：将LLM安全从单步过滤提升为轨迹级控制，更准确地捕捉持久上下文中恶意碎片的累积效应。
- **博弈论激励建模**：明确建模攻击者与控制器之间的策略互动，使选择规则具有激励一致性，降低攻击者回报。
- **三信号信任复合**：因果图、跨智能体一致性和时间漂移三者互补，比简单多数投票或单一过滤器更鲁棒。
- **细粒度自愈**：回滚基于结构风险优化，只移除可疑子图，保留已验证记忆，避免完全重置的代价。
- **实验设计严谨**：使用相同种子配对比较，消融完整，分析多维度（稳定、效用、延迟、结构、博弈、失败），结果可靠。
- **可审计性**：保留完整审计记录（候选、声明、分数、选择、恢复元数据），支持事后分析与复现。

## 八、不足与局限

- **模型与场景固定**：仅使用三个特定LLM（GPT-5.3, Llama-3.1-70B, DeepSeek-R1），未验证其他模型（如多模态、领域专用）或不同数量智能体下的泛化能力。拜占庭智能体假设仅限一个，多智能体共谋场景未评估。
- **有限轮次**：评估仅500轮，长篇部署（数千轮以上）可能暴露更慢的漂移积累或新类型不稳定。
- **图模型依赖**：因果图的构建依赖NLI和嵌入相似度，当验证上下文本身有噪声或不完整时，可能产生虚假支持或漏判。
- **计算开销**：相比单智能体基线，延迟增加约2倍（1.63e-3 vs 7.9e-4 s/token）。在延迟敏感应用中可能受限。控制器组件（声明提取、图维护、信任计算）也增加CPU开销。
- **缺失理论保证**：未提供收敛性证明或形式化的安全边界（如对抗稳健性上界）。
- **未与最新集成防御对比**：虽然提供了功能对比表，但未在相同协议设置下与Task Shield、SecInfer等最新工作定量比较，削弱了竞争力评估的完整性。
- **缺乏真实系统部署测试**：实验环境为受控仿真，未在真实MCP生态系统（如LangChain、AutoGen）中验证，未考虑网络延迟、工具调用失败等实际因素。
