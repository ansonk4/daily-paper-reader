## 摘要
大规模语言模型（LLM）智能体在广泛基准测试中取得了强劲表现，但大多数评估假设静态环境。相比之下，实际部署本质上是动态的，要求智能体持续使其知识、技能和行为与不断变化的环境及更新的任务条件对齐。为弥补这一差距，我们引入EvoArena——一个基准测试套件，将环境变化建模为终端、软件和社交领域中的渐进更新序列。我们进一步提出EvoMem——一种基于补丁的记忆范式，将记忆演化记录为结构化的更新历史，使智能体能够通过记忆中的变化推理环境演化。实验表明，当前智能体在EvoArena上表现困难，在演化的终端、软件和社交偏好领域中平均准确率仅为39.6%。EvoMem一致地提升了性能，在EvoArena上平均提升1.5%，并在GAIA和LoCoMo等标准基准上分别提升6.1%和4.8%。在单个任务之外，EvoMem还使EvoArena上的链级准确率提升3.7%，其中成功需要完成一系列连续的相关演化子任务。机制分析显示，EvoMem改善了记忆中的证据捕获，表明更好地保留了完整演化的环境状态。我们的结果强调了在评估和记忆中建模演化对于可靠智能体部署的重要性。

## Abstract
Large language model (LLM) agents have achieved strong performance on a wide range of benchmarks, yet most evaluations assume static environments. In contrast, real-world deployment is inherently dynamic, requiring agents to continually align their knowledge, skills, and behavior with changing environments and updated task conditions. To address this gap, we introduce EvoArena, a benchmark suite that models environment changes as sequences of progressive updates across terminal, software, and social domains. We further propose EvoMem, a patch-based memory paradigm that records memory evolution as structured update histories, enabling agents to reason about environmental evolution through changes in their memory. Experiments show that current agents struggle on EvoArena, achieving an average accuracy of 39.6% across evolving terminal, software, and social-preference domains. EvoMem consistently improves performance, yielding an average gain of 1.5% on EvoArena and also improving standard benchmarks such as GAIA and LoCoMo by 6.1% and 4.8%. Beyond individual tasks, EvoMem further improves chain-level accuracy by 3.7% on EvoArena, where success requires completing a consecutive sequence of related evolutionary subtasks. Mechanistic analysis shows that EvoMem improves evidence capture in the memory, indicating better preservation of complete evolving environment states. Our results highlight the importance of modeling evolution in both evaluation and memory for reliable agent deployment.

## 1. 核心问题与整体含义（研究动机和背景）
- **现有问题**：当前 LLM 智能体（Agent）评估大多基于静态环境（如固定任务、接口、规则），但真实部署中环境会持续演化——工作流依赖变化、代码库版本升级、用户偏好随时间迁移。这种“环境演化”（Environment Evolution）未得到充分建模和测试。
- **关键发现**：传统记忆系统仅维护最新状态，当环境版本变化时，旧版本相关知识会被覆盖，导致“状态崩溃”（State Collapse），智能体无法区分不同版本的有效行为。
- **研究目标**：构建可以测试智能体在环境演化下适应能力的基准，并设计一种能保留演化历史、支持版本感知推理的记忆范式。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **EvoArena 基准**：
  - 将环境变化建模为“渐进更新序列”（progressive updates），每个环境包含一条演化链（Chain），链内各版本共享同一高层次目标但操作细节不同。
  - 覆盖三个领域：
    - **Terminal-Bench-Evo**：终端工作流演化（接口、依赖、路径、权限等变更）。
    - **SWE-Chain-Evo**：软件代码库演化（里程碑式功能添加、缺陷修复，保持向后兼容）。
    - **PersonaMem-Evo**：用户偏好演化（偏好随时间改变，需隐式推断）。
- **EvoMem 记忆范式**：
  - **核心思想**：用 Git 风格的“补丁历史”（Patch History）记录每次非增量（non-additive）记忆更新。每个补丁包含：更新前状态、更新后状态、更新理由、触发证据。
  - **工作流程**：
    1. **记录阶段**：每次记忆更新时，计算当前记忆和前一时刻的差异 `Δ_t`；若为修改/覆盖操作，则生成补丁 `p_t = (时间戳, 旧内容, 新内容, 理由, 摘要, 证据)`，追加到只写补丁日志。
    2. **检索阶段**：推理时，默认从最新记忆检索；同时根据查询从补丁历史中检索相关补丁（语义+BM25混合检索），将两者拼接作为最终上下文。
  - **通用性**：适用于多种 base memory 系统（Terminus2、OpenHands、A-Mem、Memento-Skill），仅需适配补丁定义和检索接口。

## 6. 主要结论与发现
- **当前 agent 在演化环境中表现差**：
  - EvoArena 上 step 平均准确率仅 39.6%，chain 准确率更低（Terminal 21.5%、SWE 10.6%、Persona 40.0%）。
- **EvoMem 一致提升性能**：
  - EvoArena step 平均 +1.5%（终端 +2.4%、软件 +0.5%、社交 +1.8%）；chain 平均 +3.7%。
  - 标准基准：GAIA +6.1%，LoCoMo +4.8%。
  - 在 chain-level 提升比 step-level 更大，表明 EvoMem 对维持序列可靠性尤其有效。
- **机制解析**：
  - Terminal-Bench：当补丁被实际用于推理或命令时（patch uptake >0），提升最大（+8.3%）。
  - SWE-Chain：EvoMem 减少回归故障（PASS_TO_PASS 失败率降低 2.77%）。
  - PersonaMem：主要提升来自“时间轨迹”和“多模式综合”类型（+5.2%）；改善完整证据保留（row-level capture +2.4%）。
  - 增强型模型（如 GPT-5.5）能更有效利用补丁处理冲突解决（+7.6%）。
- **效率**：更高 token 消耗不一定带来更高准确率，EvoMem 可以在适度 token 下获得增益。

## 7. 优点
- **基准创新**：首次系统性地将环境演化建模为可测试的基准（EvoArena），覆盖终端、软件、社交，填补静态评估的空白。
- **方法轻量通用**：EvoMem 不改变 base memory 核心逻辑，仅增加补丁记录和检索，可快速适配多种现有 agent（Terminus2、OpenHands、A-Mem、Memento-Skill）。
- **实验严谨**：多模型、多 agent、多领域验证，包含链级和步骤级指标，以及详细的机制分析和效率分析。
- **实用价值**：揭示记忆演化对可靠部署的重要性，提供了可落地的改进方案。
