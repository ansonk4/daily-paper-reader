## 摘要
程序性记忆正日益被用于提升LLM智能体在重复性工作任务中的表现，但其生成可复用技能的能力仍缺乏深入理解。我们提出AFTER基准测试，包含382个真实企业任务，涵盖6种职业角色和22种程序性技能，旨在评估技能在任务、角色和模型骨干之间的迁移能力。该基准包含局部改进、跨任务迁移、跨角色迁移和跨模型泛化的受控评估设置。实验表明，程序性记忆在工业工作流中带来稳定提升：单次优化轮次使整体性能提高3.7-6.7个百分点，而由多样化多模型执行轨迹演化而来的技能实现了73.1%的跨模型测试准确率，优于所有单一模型轨迹来源。我们进一步发现，部分技能和任务与模型间具有广泛泛化能力，而另一些则特化于角色专属工作流，在迁移时丧失有效性。这些结果为在生产智能体平台中构建、评估和部署程序性记忆系统提供了实践指导。

## Abstract
Procedural memory is increasingly used to improve LLM agents on recurring workplace tasks, yet its ability to produce reusable skills remains poorly understood. We introduce AFTER, a benchmark of 382 realistic enterprise tasks spanning six professional roles and 22 procedural skills, designed to evaluate how skills transfer across tasks, roles, and model backbones. The benchmark includes controlled evaluation settings for local improvement, cross-task transfer, cross-role transfer, and cross-model generalization. Experiments show that procedural memory delivers consistent gains in industrial workflows: a single refinement round improves aggregate performance by 3.7-6.7 points, while skills evolved from diverse multi-model execution traces achieve 73.1% cross-model test accuracy, outperforming all single-model trace sources. We further find that some skills generalize broadly across tasks and models, whereas others become specialized to role-specific workflows and lose effectiveness under transfer. These results provide practical guidance for building, evaluating, and deploying procedural memory systems in production agent platforms.

## 1. 核心问题与整体含义

### 研究背景
- LLM智能体 increasingly 被用于企业环境中处理重复性任务（如文档处理、数据库查询、基础设施配置等）
- **程序性记忆（Procedural Memory）** 是一种从过往执行轨迹中提取的可复用指令/策略层，用于提升智能体在重复任务上的表现
- 然而，程序性记忆能否真正跨任务、跨角色、跨模型骨干迁移，这一问题缺乏系统性理解

### 核心问题
- 现有基准和系统存在两个关键缺陷：
  - 将**局部改进（local improvement）** 与**真正的迁移（true transfer）** 混为一谈
  - 缺乏受控环境来测试程序性知识在何种条件下可复用，何时仅是对源环境的过拟合
- 关键张力：**专业化（specialization）vs. 泛化（generalization）** —— 技能应针对特定工作流优化，还是从多样化经验中学习以获得广泛迁移能力？

### 整体含义
- 将程序性记忆从"静态提示工件"重新定义为**智能体能力的可进化层**，可以在真实工作条件下学习、控制和评估

**EVOLUTION框架的循环流程：**
1. **COLLECT**：使用当前技能执行源任务，收集执行轨迹

3. **REVISE**：基于当前技能和诊断，提出候选技能体
4. **PROMOTE/ROLLBACK**：若候选版本在验证集上提升超过阈值δ，则采纳；否则保留为 inactive 分支

## 6. 主要结论与发现

### 核心发现
1. **静态技能价值**：
   - 技能平均提升完全通过率 +2.8 百分点
   - 弱模型受益更显著（如Gemma 4 E4B在GenAI角色提升+14.2）
   - LLM生成技能（G）常优于人工编写技能（H）

2. **单轮优化效果**：
   - 单轮LLM引导优化带来 +3.7 至 +6.7 个百分点的整体提升
   - 较大模型受益更稳定，Infra和SWE角色提升最大

3. **框架引导进化**：
   - 所有5个框架在泛化上均面临挑战
   - 训练集上的大提升不一定迁移到held-out任务
   - 窄经验（n=1）常导致过专业化，多样经验（n=8.3）在某些框架中带来更好迁移

4. **跨模型迁移**：
   - 多模型多样化痕迹进化的技能达到 **73.1%** 跨模型测试准确率
   - 优于最佳单模型来源（+13.7百分点）
   - **意外发现**：较弱源模型提供比更强模型更好的可迁移信号

5. **跨角色迁移**：
   - 技能自然特化于本地工作流
   - 角色间迁移可能导致性能下降（-4.8至-7.5百分点）
   - 不同角色使用同一技能的目的不同（如pdf提取：PM用于执行摘要，DS用于数据摄入）

6. **Token效率**：
   - 进化技能通过将程序性知识前置到提示中，减少推理时Token消耗
   - Claude在Kafka Lag Anomaly Detection任务上减少326k Token（62%）

## 8. 不足与局限

### 基准覆盖局限
- **领域局限**：仅覆盖技术组织中的角色，未涵盖医疗、法律、科研等领域
- **技能局限**：22种技能排除了开放式创造性或对话性任务
- **任务来源偏差**：部分任务来自作者自身实践，可能引入领域偏好

### 评估范围局限
- **痕迹预算固定**：实验固定了每次进化运行的痕迹数量，实际部署可能积累更大规模的痕迹池
- **验证器局限**：使用pytest自动化验证，仅测量功能正确性，未覆盖代码可读性、边缘案例鲁棒性、用户偏好等质量维度
- **跨角色实验不足**：仅评估了3项高重叠技能的跨角色迁移，结论普适性存疑

### 模型与框架选择
- **非穷尽评估**：部分前沿模型和记忆系统因API访问限制或发布时间未能纳入
- **框架对比仅用3项技能**：可能无法全面反映各框架的优劣
- **跨模型迁移实验仅使用单一求解器**：结论在不同求解器上的稳定性未验证

### 方法论局限
- **未报告置信区间**：实验结果缺乏不确定性量化
- **进化轮次有限**：主要评估单轮优化，多轮迭代的效果和潜在退化未充分探讨
- **技能表示单一**：仅使用文本SKILL.md格式，未探索结构化或代码化的技能表示

### 潜在偏差
- **LLM生成任务的质量**：288个任务由Claude Sonnet 4.6起草，可能存在模型特定的风格偏好
- **评估模型与进化模型的重叠**：部分实验使用了参与框架运行的模型作为评估目标，可能引入偏差
