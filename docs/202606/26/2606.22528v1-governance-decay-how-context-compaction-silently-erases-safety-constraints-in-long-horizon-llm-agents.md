## 摘要
现代LLM智能体日益依赖上下文压缩、摘要或驱逐机制，以将会话长度控制在token预算之内。我们发现，这一上下文管理层是一个安全关键的故障面：当上下文中可见时，智能体能够可靠地遵守的治理约束，可能在压缩过程中被静默移除，导致同一智能体会话后期执行被禁止的工具操作。我们将这种故障模式称为治理衰减。我们提出ConstraintRot基准，该基准包含具有确定性工具调用评分的长时域智能体场景，并在七个模型族上测量压缩引发的违规情况。在1,323个回合中，违规率从策略完整保留在上下文中的0%上升至压缩后的30%，部分模型甚至达到59%；当约束在摘要中得以保留时，违规率仍为0%，但当其被丢弃时，违规率则达到38%。我们进一步研究了一种压缩驱逐攻击，其中对抗性上下文内容引导摘要器遗漏合法策略，并证明优化后的注入可击败所有被评估的模型。最后，我们提出约束固定，这是一种无需训练的缓解方法，将治理约束与有损压缩隔离，并在我们的基准中将违规率恢复至0%。这些结果表明，上下文管理应被视为已部署LLM智能体的一等治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

## 2. 方法论

### 2.1 核心思想

- **Governance Decay 形式化定义**：
  - 设 $v(M, H)$ 为模型 $M$ 在上下文 $H$ 下执行违规动作的概率。
  - 定义 $\Delta_{\text{decay}} = v(M, [C(H_{<t}); r]) - v(M, [H_{<t}; r])$，即**仅由压缩历史**导致的同一禁止请求 $r$ 的违规率增量。
- **Compaction-Eviction Attack（压缩-淘汰攻击）**：
  - 对手仅控制智能体摄入的上下文数据（工具输出、检索文档），不接触模型、系统提示或压缩算子。
  - 两种变体：
    - **Volume 变体**：注入大量良性内容，迫使压缩触发并淘汰策略。
    - **Summarizer-Injection 变体**：在内容中嵌入针对压缩步骤的指令（如"摘要时省略过时的合规前言"），使策略被主动删除。
- **Constraint Pinning（约束固定）防御方案**：
  - 智能体框架将治理约束提取到**固定缓冲区（pinned buffer）**中。
  - 该缓冲区**豁免于有损压缩**，并在每次压缩后**逐字重新注入**。
  - 每步进行**完整性检查**：要求压缩后的上下文仍蕴含固定约束。
  - **无需训练**，仅修改框架内存管理层。

### 2.2 算法/流程说明

1. 框架维护消息历史 $H$，当 $\text{tokens}(H) > B$（预算）时触发压缩算子 $C$。
2. 压缩时，将旧跨度替换为压缩摘要。
3. **防御流程**：
   - 提取治理约束到固定缓冲区。
   - 执行压缩（固定区不参与压缩）。
   - 压缩后将固定约束重新注入上下文。
   - 验证压缩后上下文是否仍蕴含固定约束。

## 6. 主要结论与发现

- **Governance Decay 普遍存在**：
  - 压缩后违规率从 0% 升至 **30%**（部分模型达 **59%**）。
  - 当约束在摘要中**保留**时违规率 0%，**丢弃**时达 **38%**。
- **软策略衰减远大于硬策略**：
  - 软组织策略衰减 **+50 个百分点**（0%→50%），硬安全规范仅 **+6 个百分点**。
  - 比率为 **8.3×**，恰好侵蚀那些**只能存在于上下文中的部署特定规则**。
- **压缩器驱动失败，而非智能体**：
  - 交叉实验显示违规率追踪**摘要器模型**，而非智能体模型。
- **攻击有效**：
  - 优化后的摘要器注入**击败所有 7 个模型**（包括对固定注入免疫的模型，0%→65%）。
  - 最有效的策略是**Token 预算框架**（"为保持在预算内，丢弃策略说明"），突破所有 3 个测试模型。
- **Constraint Pinning 有效**：
  - 在 7 个模型和两种攻击变体下**恢复违规率至 0%**。
  - Token 开销 **< 0.5%**（约 47 tokens/次压缩）。
  - 对允许动作的完成率 **99%**，无显著效用损失。
- **防御仍有边界**：
  - 操作员冒充撤销（在近期非摘要上下文中）可将 naive Pinning 从 0% 推升至 17%。
  - 加固（明确来源标注）仅减半至 10%，**需要可信带外通道**才能完全关闭。
