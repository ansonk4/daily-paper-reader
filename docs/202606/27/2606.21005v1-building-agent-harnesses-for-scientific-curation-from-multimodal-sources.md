## Abstract
Scientific discovery workflows often depend on structured curation from the literature. This is difficult for current agents because the key evidence is scattered across long text, dense tables, and figures, and the final records often require reasoning across multiple evidence fragments rather than copying a single span. We study scientific curation from multimodal sources and introduce Beaver, an agent harness that extracts structured information from scientific papers while preserving provenance to the supporting evidence. Beaver combines a frontier agent with multimodal evidence tooling, task scaffolding, and artifact-grounded autoresearch. These components turn curation into a staged, auditable workflow and enable an iterative evaluate--diagnose--revise loop, where persistent run artifacts expose stage-localized failures and guide harness updates. Experiments show that Beaver reaches 81.0 on Gold-Referenced Attribute Score (GRAS), an attribute-level measure of agreement with gold curated records, outperforming frontier agents by over 23 absolute points. Ablations show that task scaffolding, multimodal evidence tooling, and provenance traces each contribute meaningfully to performance, while attribute-level analysis shows the largest gains on high-value attributes that require cross-modal reasoning and normalization. These results show that, for scientific curation from papers with multimodal evidence, harness design is a central determinant of agent performance.

## 2. Proposed Methodology

### 2.1 Core Idea

- Beaver is designed around the premise that **harness design**, not just model capability, is the central lever for multimodal scientific curation.
- It turns curation into a **staged, auditable workflow** with an **evaluate–diagnose–revise loop**:
  1. A base frontier agent provides general coding and reasoning capability.
  2. Multimodal evidence tooling exposes text, tables, and figures in machine-friendly forms.
  3. Task scaffolding decomposes the end-to-end curation into ordered stages.
  4. Artifact-grounded autoresearch iteratively improves the harness using persistent run artifacts and stage-localized diagnostics.

### 2.2 Task Formulation

- For a paper \(p\), inputs are:
  - \(T(p)\): textual content
  - \(B(p)\): tables
  - \(G(p)\): figures
  - \(F\): canonical attribute set defined by the curation schema
- The output is:
  - A predicted table \(\hat{Y}\) whose columns are drawn from \(F\)
  - A provenance artifact \(\hat{C}\) linking each extracted value back to supporting evidence in the paper
- The task is therefore **schema-constrained table construction**, not free-form QA or single-span extraction.

### 2.3 Harness Components

#### Base Agent

- Beaver sits on top of a **frontier coding agent** that can read, modify, and execute code in a local environment.
- The base agent alone is insufficient: it performs substantially worse than the full Beaver harness, which shows that the improvement comes from the surrounding harness, not from swapping to a stronger model.

#### Multimodal Evidence Tooling

- Beaver equips the agent with an auxiliary tool set \(U\) that produces and operates on derived artifacts:
  - **Paper-to-markdown transformation**: each article is converted into an outline plus section-level markdown files, making long documents easier to navigate than raw XML/PDF.
  - **Parsed table viewer**: helps the agent recover quantitative values from tables.
  - **Calibrated gridline overlay tool**: assists in extracting numeric values from plots by calibrating visual axes and reading off data points.
- These tools do not solve extraction directly, but they provide a better interface to textual, tabular, and visual evidence, enabling more accurate scaffolded actions.

#### Task Scaffolding

- Rather than asking the agent to solve the entire curation problem in one pass, Beaver decomposes extraction into **sequential stages** from easier paper-level attributes to harder arm-level and endpoint-level attributes.
- Each stage is not only an attribute subset but also a **workflow scaffold** that guides the agent through:
  1. Preprocessing relevant documents, tables, and figures
  2. Navigating to candidate evidence
  3. Multimodal reasoning and annotation over text, tables, and plots
  4. Extracting values with provenance traces
  5. Normalizing entities and units into the schema
  6. Reviewing and refining the output
- Formally, stage \(k\) predicts:
  \[
  \hat{Y}^{(k)} = h_k\left(p, A(p), \hat{Y}^{(k-1)}; \omega_k, U\right)
  \]
  where \(\hat{Y}^{(0)}\) is empty, \(\omega_k\) specifies the stage workflow, and \(U\) exposes the evidence tools.
- Later stages consume carried-forward outputs of earlier stages, which reduces ambiguity and narrows the search space.
- This staged formulation is also crucial for iteration: it allows the harness to **diagnose and revise failures at a specific stage** without conflating unrelated errors elsewhere.

#### Artifact-Grounded Autoresearch

- The base agent, multimodal evidence tools, and task scaffolding together form the **initial harness** \(H_0\).
- Beaver then improves this harness through **persistent filesystem state** and an **evaluate–diagnose–revise loop**.
- Unlike free-form self-improvement, Beaver uses the **stage structure** from task scaffolding to localize failures.

**Algorithm flow (high-level prose description):**

1. For each outer iteration \(t = 0, 1, \dots, T-1\):
   - Evaluate the current harness \(H_t\) on a development set \(D_{\text{dev}}\).
   - Decompose the evaluation by stage using the task scaffold.
   - Select the **lowest-performing stage** \(k^*\).
   - Initialize a **stage-specific workspace** containing progress tables, logs, run summaries, raw outputs, agent trajectories, provenance traces, and other artifacts.
2. For each inner iteration \(j = 1, \dots, J\):
   - Reset the working harness to the current best checkpoint.
   - Review the stage workspace and **propose targeted changes** to the harness.
   - Apply a candidate patch to the harness (updating scaffold, tools, scripts, prompts, validators, etc.).
   - Run the focused stage on \(D_{\text{dev}}\) with the updated harness.
   - Update the filesystem with new results, logs, trajectories, and summaries.
   - Accept the revision as the new best harness only if it improves stage performance; otherwise discard it.
3. After the inner loop, proceed to the next outer iteration with the best accepted harness.

- This design yields a **two-level optimization**:
  - Outer loop: select worst stage.
  - Inner loop: propose/test/retain patches for that stage using grounded artifacts.
- Because the loop is stage-localized and artifact-grounded, it avoids accumulating unsuccessful patches and builds on prior evidence without destabilizing other stages.

### 2.4 Evaluation Metrics

- The paper introduces **Gold-Referenced Attribute Score (GRAS)**:
  1. For each paper \(p\), let \(\hat{R}(p)\) be predicted rows and \(R^\star(p)\) be gold rows.
  2. Align predicted and gold rows within the same paper using **maximum-weight bipartite matching** under an attribute-aware row-similarity score.
  3. Compute attribute scores with type-specific functions:
     - Exact-match attributes: indicator \(1[\hat{r}_f = r^\star_f]\)
     - Lexical attributes (e.g., drug names): token-level F1
     - Numeric attributes (e.g., durations, outcome values): relative accuracy \( \max(0, 1 - |\hat{r}_f - r^\star_f| / |r^\star_f|) \)
  4. For each aligned gold row \(r^\star\), compute its row score as the mean attribute score over evaluated attributes.
  5. GRAS averages over all gold rows:
     \[
     \text{GRAS} = \frac{1}{\sum_p |R^\star(p)|} \sum_p \sum_{r^\star \in R^\star(p)} S(\pi(r^\star), r^\star)
     \]
     where \(\pi(r^\star)\) is the matched predicted row or null if unmatched.
- Unmatched gold rows remain in the denominator and incur **one-sided missingness penalties**, so GRAS jointly reflects row recovery and attribute correctness while staying anchored to professionally curated reference rows.

## 3. Experimental Design

### 3.1 Datasets and Benchmark Construction

- Existing benchmarks do not match the target setting: full-paper, multimodal, rich production-style schema, expert-curated reference, long-horizon extraction.
- The authors construct a new benchmark from a **professionally curated PubMed corpus** in Alzheimer’s disease:
  - 425 references
  - 327 studies
  - 929 study arms
  - 106,257 patients
  - 58,058 curated data rows
  - 143 interventions, 69 efficacy endpoints, 373 safety endpoints
- The curation schema has **over 400 attributes**, organized around identifiers, treatment descriptors, covariates, endpoint metadata, and reported outcome values.
- For the main experiments, they select **20 high-value attributes** for evaluation.

### 3.2 Evaluation and Development Sets

- Evaluation is performed on a selected set of **23 papers** from PMC Open Access, chosen because full-paper access is available for every article.
- For harness development, they use a smaller **development subset of 8 papers** sampled from the full evaluation set, enabling faster iteration while still covering representative failure modes and stable anchor cases.

### 3.3 Baselines and Compared Methods

- Four harness baselines are compared:
  - **Codex CLI**
  - **Claude Code**
  - **Copilot CLI**
  - **Meta-Harness** (state-of-the-art open-source harness on Terminal Bench 2.0)
- Frontier LLMs used include:
  - GPT-5.4
  - Claude Opus 4.7
  - Claude Opus 4.6
  - Claude Sonnet 4.6
  - Claude Haiku 4.5
- Beaver itself uses **Copilot CLI as the base agent** and **GPT-5.4 as the backend model**.
- This setup focuses the comparison on **harness design under realistic high-capability model conditions**, not on differences caused by weak base models.

### 3.4 Training / Autoresearch Configuration

- Task scaffolding decomposes each task into **4 stages**.
- For artifact-grounded autoresearch:
  - Outer iterations: \(T = 3\)
  - Inner-loop iterations: \(J = 10\)
  - Total: **634 iterations**
- Each harness is run **three times**, and mean and standard deviation are reported, because agent executions exhibit meaningful stochastic variance.

## 6. Main Conclusions and Findings

- Beaver achieves **81.0 GRAS**, while the strongest baseline reaches only **58.6%**, a **22.4-point absolute improvement** over Codex with the same GPT-5.4 backend.
- Compared with Copilot CLI (same base agent), Beaver isolates a **23.6-point gap**, showing that the improvement comes from harness design rather than a different agent interface.
- Simply swapping to another high-capability frontier agent (e.g., Claude Opus) is **not sufficient** to close the gap.
- Meta-Harness performs substantially worse than Beaver despite using the same GPT-5.4 backend, indicating that **generic harness optimization does not directly solve multimodal scientific curation**.
- Attribute-level analysis shows Beaver improves across multiple high-value attributes, with the largest gains on:
  - `endpoint.unit`
  - `change`
  - `percentchange`
  - `value`
  - `n.observed`
  These are attributes that require **cross-modal reasoning, table/figure access, and schema-aware normalization**.
- Ablations show all three components contribute substantially:
  - Full Beaver: 81.0
  - No provenance traces: 70.7
  - No multimodal evidence tooling: 66.1
  - No task scaffolding: 60.5
- Removing task scaffolding causes the largest drop, followed by removing multimodal evidence tooling, then removing provenance traces.
- Compute–performance analysis shows Beaver sits at the top of the empirical frontier, converting agent compute into curation quality more efficiently than ablated variants.

## 7. Strengths in Method or Experimental Design

- **Task realism.**  
  The benchmark reflects a genuinely hard, production-style curation setting: full papers, multimodal evidence, rich schema, and expert-curated gold references.
- **Harness-as-object-of-optimization view.**  
  The paper clearly demonstrates that **harness design is a first-class determinant of performance**, not just model choice.
- **Staged, auditable workflow.**  
  Task scaffolding makes the process interpretable and debuggable, and enables stage-localized improvement.
- **Artifact-grounded autoresearch.**  
  Using persistent filesystem artifacts and stage-localized diagnosis is a principled way to iteratively improve harnesses without destabilizing the whole pipeline.
- **Provenance traces.**  
  Even though not directly scored, provenance improves auditability and indirectly supports better autoresearch decisions.
- **Evaluation metric design.**  
  GRAS is attribute-aware, gold-referenced, and penalizes unmatched gold rows, aligning well with the real-world need for complete and correct curation.
- **Thorough ablations and failure analysis.**  
  The paper provides component-level ablations, compute–performance frontiers, and detailed case studies of both successes and remaining failures.

## 8. Weaknesses and Limitations

- **Domain coverage.**  
  The benchmark is focused on **Alzheimer’s disease** only. While this provides clinical and structural breadth within that domain, it leaves open how well the approach generalizes to other disease areas, document styles, or schema structures.
- **Evaluation set size.**  
  The 23-paper evaluation set is moderate. This is acceptable for an initial study but may limit the strength of broad statistical claims.
- **Schema and domain specificity.**  
  The curation schema is highly specialized and professionally curated. This is a strength for realism but may limit direct transfer to other scientific domains without similar expert-annotated resources.
- **Compute cost and accessibility.**  
  The autoresearch loop requires **hundreds of millions of tokens** and many iterations. While this is reported and analyzed, it may still be a barrier for smaller teams or organizations without access to frontier models and large inference budgets.
- **Remaining failure modes.**  
  Beaver still struggles with:
  - Repeated analysis-population variants (e.g., FAS, observed-cases, completers), where it collapses multiple rows into one per subgroup-endpoint.
  - Some fine-grained schema axes that are not yet treated as row-distinguishing fields.
- **Limited baselines in same harness family.**  
  The paper compares against strong frontier agents and one generic harness optimization method, but there is less exploration of other domain-specific harness designs or alternative multimodal extraction pipelines.
- **Human review not deeply integrated.**  
  The paper argues for human review and auditability as part of responsible use, but does not present a formal user study or human-in-the-loop evaluation of how provenance and staging interact with real curators.
- **Generalization of autoresearch.**  
  The autoresearch loop is shown to help, but the paper does not fully analyze how well the discovered harness changes transfer across document types, schemas, or domains.
