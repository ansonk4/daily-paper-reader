## 摘要
智能体记忆正在转向图结构，而目前为它构建的溯源防御都只检查一件事：智能体检索到的记录的来源。我们表明，整个这类防御在构造上是盲目的。长期图记忆在可写的图结构上运行全局选择步骤，因此不受信任主体写入的结构会改变选择哪些经过认证的事实，而被引用的证据仍然完全经过认证；忠实的信息流控制（IFC）会检查读者使用的所有信息的来源（所有信息都经过认证），但在文档-QA基准和真实的多会话智能体记忆上，它做出与毫无防御完全相同的字节级决定。在最关键的例子中，一个无来源的结构性写入在499个真实操作中悄然误导了28次不可逆的账本转移：忠实的IFC允许每一次，而\authselect\阻止了每一次。然后我们精确刻画了哪些记忆暴露了：当选择器的结构项可以重新分配一个Ω(1)份额的top-k成员资格超过所选事实的边界时，它就接纳了该信道。个性化PageRank可以做到，因为无来源写入会重新路由守恒的随机游走质量；固定内容的重新排序器则无法做到，而Graphiti的节点距离（其依赖结构比PageRank更甚）仍然免疫。可重分配性，而非依赖性，才是预测因子。我们一般性地证明了免疫情况，并在我们验证的瓶颈条件下证明了开放情况。关闭信道迫使任何溯源防御在认证子图上重新计算选择，这正是\authselect\所做的，开销为零区块且延迟增加2-3%。

## Abstract
Agent memory is moving to graphs, and the provenance defenses now being built for it all check one thing: the provenance of the records an agent retrieves. We show that this entire class of defense is blind by construction. A long-term graph memory runs a global selection step over writable graph structure, so structure that an untrusted principal writes changes \emph{which} authenticated facts are selected while the cited evidence stays fully authenticated; faithful information-flow control (IFC), checking the provenance of what the reader uses (all of it authenticated), makes the byte-identical decision to no defense at all, across document-QA substrates and real multi-session agent memory. In the most consequential instance, a no-source structural write silently misdirects $28$ irreversible ledger transfers over $499$ live actions: faithful IFC permits every one, and \authselect\ prevents every one. We then characterize exactly which memories are exposed: a selector admits the channel when its structural term can reallocate an $Ω(1)$ share of top-$k$ membership past a selected fact's margin. Personalized PageRank can, since a sourceless write reroutes conserved random-walk mass; a content-fixed reranker cannot, and Graphiti's node-distance, which leans on structure \emph{more} than PageRank does, stays immune. Reallocatability, not reliance, is the predictor. We prove the immune case in general and the open case under a chokepoint condition we verify. Closing the channel forces any provenance defense to recompute selection on the authenticated subgraph, which is what \authselect\ does, at zero over-block and $2$--$3\%$ latency.

# 论文总结：Selection Integrity for LLM Graph Memory: An Accumulability Criterion for Information-Flow-Blind Retrieval

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：LLM 智能体的长期记忆正转向图结构（如 HippoRAG、Graphiti），记忆通过全局选择步骤（如 PageRank、社区划分）从可写的图结构中检索相关事实。现有安全防御（如 A-MemGuard、SuperLocalMemory、MemoryGraft）均只检查**检索到的记录本身**的溯源（provenance），即验证记录来源是否可信。
- **核心问题**：图记忆的选择步骤是对可写图结构的全局计算。不受信任的主体（如被攻破的 peer 智能体、被投毒的工具结果）可以提交**无来源的结构性写入**（如边、合并链接、关系断言），这些写入本身不是可检索的记录，但会改变哪些**已认证的事实**被选择出来。由于被引用的证据全部是已认证的，忠实的信息流控制（IFC）无法检测这种操纵——它做出与无防御完全相同的字节级决策。
- **整体含义**：现有溯源防御在构造上对这类**隐信道**（stealth channel）是盲目的。攻击者可通过操纵图结构实现对抗性记忆操控，例如在 499 个实际动作中悄然误导 28 次不可逆的账本转账。这暴露了图记忆安全的一个全新维度：选择完整性（selection integrity）必须得到保护。

## 2. 提出的方法论：核心思想、关键技术细节

### 2.1 累积性准则（Accumulability Criterion）
- **核心思想**：并非所有依赖图结构的选择器都受该信道影响。信道是否开启取决于选择器的结构信号能否**重新分配 top-k 成员资格**（reallocatability）。
- **形式化**：设选择器为 `s(i) = g(c(i,Q), σ(i;G,Q))`，其中 `c` 为内容项，`σ` 为结构项，`g` 在 `σ` 上严格递增。信道开启当且仅当结构项满足：
  - (a) **成员可重分配性**：无来源的结构写入可以将决定 top-k 的 Ω(1) 份额的质量重分配到不在任何返回项推导闭包内的节点。
  - (b) **边界跨越**：重分配的质量超过目标项的锚定边界，从而改变选择集。
- **实例验证**：
  - **Personalized PageRank (PPR)** 满足条件 (a)，因为无来源边会重新路由守恒的随机游走质量。低种子、多跳的目标（teleport floor 低于捕获界）易被挤出。
  - **固定内容的重新排序器**（如向量检索 + BM25 + 有界重排）不满足条件 (a)，因为候选集由内容固定，结构写入无法引入新成员。
  - **Graphiti 的节点距离**重度依赖结构，但因其候选集固定、重排有界，条件 (a) 失败，因此免疫。
- **预测因子**：可重分配性（`Sadv`），而非结构依赖度（`Sdom`）。实验表明节点距离的 `Sdom=0.63`（比 PPR 的 `0.24` 更依赖结构），但其 `Sadv≈0`，即无法被对抗性利用。

### 2.2 防御方法：AUTH SELECT
- **原理**：在认证子图上重新执行原生选择器，比较全图结果与子图结果。若结果不变则接受，否则使用子图结果或按权限升级。
- **算法流程**：
  1. 在全图 `G` 上运行选择器，得到结果 `a_G`。
  2. 移除所有未经认证的结构写入（`W_U`），得到认证子图 `G_auth`。
  3. 在 `G_auth` 上重新运行**相同**的选择器，得到结果 `a_{G_auth}`。
  4. 若 `a_G` 与 `a_{G_auth}` 一致，则接受 `a_G`；否则使用 `a_{G_auth}` 或根据动作权限升级。
- **关键设计原则**：
  - 完整性跟随写入来源（write provenance），而非端点标签。即使边连接的是两个已认证的节点，若边来自不受信任的源，仍被视为不可信。
  - 重放的是原生选择器，不替换为其他算法。
  - 分歧被视为权限事件，而非直接拒绝。

## 6. 主要结论与发现
- **存在性**：图记忆中存在着 IFC 盲信道。无来源结构性写入可改变已认证事实的选择，而忠实 IFC 与无防御​​结果相同。
- **累积性准则**：信道只会在结构项能重分配 top-k 成员资格的选择器上开启（如 PPR、Leiden 社区），而内容固定或结构性有界的选择器（如 RRF、节点距离、纯向量检索）免疫。
- **预测因子**：可重分配性（`Sadv`）而非结构依赖度（`Sdom`）是信道开启的预测因子。
- **必要性**：任何只检查检索记录来源的后选择防御（如理由 IFC、Bayesian trust、rerank）都无法关闭信道。唯一可行的方法是在选择之前移除不可信结构并重选，即选择完整性。
- **有效性**：AUTH SELECT 在所有开启信道的实验上实现零伤害（harm=0）、零过度阻断（over-block=0），延迟增加仅 2-3%（额外一次 PPR 传播）。
- **能力边界**：注入-only 攻击（即使携带内容）无法在本实验设置中开启隐信道（0/35 个隐蔽有害案例），因为足够强的注入会变得可检索因而可见。结构性写入是解耦杠杆与可见性的关键能力。
- **实时危害**：在 499 个实际动作中，无来源写入导致 28 次不可逆的转账误导；AUTH SELECT 阻止全部。

## 7. 优点
- **新颖性**：首次指出图记忆选择步骤的 IFC 盲信道，并提出选择完整性的概念。与现有仅检查记录溯源的工作有本质区别。
- **理论与实践结合**：提出可验证的累积性准则（数学定义 + 实验验证），并有证明（Proposition 1、Lemma 1/2、Theorem 1/2）。准则能正确预测哪些选择器开启信道，哪些免疫。
- **清晰的威胁模型**：区分注入-only 与结构性写入两种能力，并论证结构性写入在现实部署（多智能体共享记忆、工具结果导入、直接 KG 损坏）中的可行性。
- **高效的防御**：AUTH SELECT 实现简单，无需修改底层选择器，开销极低（零过度阻断、2-3% 延迟），且理论上紧致（不可约性推论说明任何信道关闭防御都等价于选择完整性）。
- **广泛实验**：覆盖多个系统、多个选择器、多个数据集（包括真实多会话记忆）、多种攻击变体（oracle、金无关、自适应、注入-only）、多种防御（后选择、前选择、生产级），并包含消融和稳健性检验（随机 reader、权重异常、溯源召回）。

## 8. 不足与局限
- **实验规模**：知识图谱 QA 数据集（HotpotQA 子集有 794 个段落）并非 web 规模；实验在中等规模下进行。社区划分实验（Leiden）仅有 38 个目标，样本量中等。部分结论在更大规模的图上可能变化。
- **泛化性**：仅测试了 PPR、Leiden、RRF、节点距离、向量检索几种选择器。Graphiti 的图模式（Mem0g）未测试。累积性准则对其他更复杂的选择器（如学习到的图神经网络）是否适用尚待验证。
- **可用性权衡**：当不可信结构中包含真正有用的信息（如来自低信誉源的辅助记忆）时，AUTH SELECT 会移除它从而降低效用。作者仅在 n=9 的良性不可信子集上测量了效用保留（utility 1.0 vs 暴力移除 0），样本很小。实际部署中需要更精细的权限路由。
- **理论假设**：证明中假设读者在固定选择集下是确定性的，且所有未认证影响要么是可见内容要么是结构性扰动。实际中可能存在其他影响路径（如通过元数据泄露）。对 PPR 的开放侧证明依赖于瓶颈条件（A-chokepoint），虽然作者在实验中验证了该条件，但理论上可能在某些图结构中不成立。
- **资源报告不完整**：未报告 GPU 型号、数量、运行时长，不利于复现成本评估。
- **现实危害范围**：测得的危害仅限 QA 答案错误、工具调用决策和沙盒账本转账，未涉及真实不可逆世界效应。作者也承认这一点（scope note）。
- **对注入-only 的适用性**：论文明确说明 AUTH SELECT 的价值在于无来源结构性写入的领域，而非所有记忆投毒。在注入-only 场景下，现有内容防御已足够，AUTH SELECT 帮助有限（但替代方案 rebuild-from-text 仍不如选择完整性）。
- **标签来源假设**：防御依赖于准确的写入溯源标签。在部分溯源场景（如只能发现 60% 的不可信写入）下，伤害会逐渐上升但仍低于无防御；完全溯源是理想化假设。作者认为无来源写入在实践中携带完整标签，但更复杂的溯源丢失场景需进一步评估。
