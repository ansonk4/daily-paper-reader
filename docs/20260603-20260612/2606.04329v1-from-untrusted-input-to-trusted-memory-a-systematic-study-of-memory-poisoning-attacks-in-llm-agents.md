## 摘要
内存是AI Agent的核心组件，使其能够在交互中积累知识并提升性能。然而，持久性内存引入了内存投毒的风险，即单次对抗性内存写入可能对Agent行为产生长期影响。我们对基于LLM的Agent中的内存投毒进行了系统研究。我们识别出四种内存写入通道和九种结构漏洞，涉及模型能力、系统提示设计和Agent系统架构，这些漏洞使得这些通道可被利用。基于这些漏洞，我们发展出六类内存投毒攻击的分类。此外，我们设计了MPBench——一个用于评估内存投毒攻击的基准，并表明被设计为更积极地写入和检索内存的Agent更易被利用。我们还发现现有的提示注入防御无法涵盖内存投毒攻击。我们的发现为理解和缓解针对AI Agent的内存投毒攻击提供了基础。

## Abstract
Memory is a core component of AI agents, enabling them to accumulate knowledge across interactions and improve performance. However, persistent memory introduces the risk of memory poisoning, where a single adversarial memory write can exert long-term influence over agent behavior. We present a systematic study of memory poisoning in LLM-based agents. We identify four memory write channels and nine structural vulnerabilities in model capabilities, system prompt design, and agent system architecture that make these channels exploitable. Based on these vulnerabilities, we develop a taxonomy of six classes of memory poisoning attacks. Furthermore, we design MPBench -- a benchmark for evaluating memory poisoning attacks, and show that agents designed to write and retrieve memory more aggressively are more exploitable. We also show that existing prompt injection defenses fail to cover memory poisoning attacks. Our findings provide a foundation for understanding and mitigating memory poisoning attacks against AI agents.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：LLM Agent中的持久性内存（long-term memory）引入了一种新的安全风险——**内存投毒（memory poisoning）**。攻击者只需通过一次成功的对抗性写入，就能长期操纵Agent的未来行为，而无需每次交互都注入恶意指令。
- **研究动机**：现有工作分散且缺乏系统性，威胁模型假设过强（如要求直接内存访问），或仅评估单次会话内的攻击（如提示注入）。没有统一的漏洞分析和防御设计基础。
- **整体含义**：内存是Agent能力提升的关键，但同时也是攻击面。本研究首次系统性地分析了Agent内存系统的漏洞、攻击分类和防御盲区，为后续防御设计提供了理论基础。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：从Agent内存写入的**触发机制**和**内容决策权**出发，识别四个内存写入通道（C1–C4），然后分析这些通道被利用所需的结构性漏洞（V-M1–V-M5, V-P1–V-P2, V-S1–V-S5），最后基于漏洞制定六类攻击。
- **四个内存写入通道**：
  - **C1: 显式指令执行写入（Explicit instruction-executed write）**：外部输入直接包含写内存指令，Agent无条件执行。
  - **C2: 系统提示驱动写入（System prompt-driven write）**：系统提示中定义了写内存策略（如“存储重要信息”），Agent根据自身判断写入。
  - **C3: 压缩驱动写入（Compaction-driven write）**：当上下文窗口达到限制或会话结束时，Agent对历史进行压缩总结并写入持久化内存。
  - **C4: 经验到过程写入（Experience-to-procedure write）**：Agent在完成一个任务后，将整个执行轨迹合成一个可复用的程序化技能（skill）存入过程性内存。
- **九种结构漏洞**：
  - **模型能力层面（V-M）**：V-M1（指令-数据边界盲区）、V-M2（多源上下文中的来源归因失败）。
  - **系统提示层面（V-P）**：V-P1（内存写入策略过于模糊）、V-P2（压缩时不区分信任/非信任来源）。
  - **系统架构层面（V-S）**：V-S1（写路径无验证）、V-S2（共享多源上下文）、V-S3（可操纵的压缩触发器）、V-S4（技能创建无验证）、V-S5（自我改进的放大效应）。
- **攻击分类**（六类）：
  - **强信号攻击**：显式命令插入（Explicit Command Insertion）、条件命令插入（Conditional Command Insertion）、显著性驱动的压缩投毒（Salience-driven Compaction Poisoning）。
  - **弱信号攻击**：策略符合性事实注入（Policy Conformant Fact Injection）、虚假先例插入（False Precedent Insertion）、技能-过程插入（Skill-Procedure Insertion）。
- **威胁模型**：黑盒外部攻击者，无特权访问，不知道模型和内存内容，只能通过外部输入（网页、文档、工具输出等）注入恶意内容。
- **攻击成功的条件**：触发内存写入 → 控制写入内容 → 在后续会话中触发检索并影响行为。

## 3. 实验设计：数据集、Benchmark、对比方法
- **Benchmark**：MPBench，专门评估内存投毒攻击，包含两个阶段：内存写入（ASR）和跨会话检索（RSR）。
- **数据集**：
  - 共 **3,240 个测试用例**，覆盖六个攻击类别（每类240–600个）。
  - 另外 **2,997 个良性示例**用于评估假阳性率（FPR）。
  - 任务场景涵盖 **7 个领域**：文件操作、网页浏览、邮件、日历、Slack、脚本执行、技能调用。
  - 交付模式：静态上下文（直接提供有效载荷）和动态工具调用（Agent主动检索文件内容）。
- **Agent系统**：OpenClaw（Steinberger, 2025）和 HERMES（Nous Research, 2025）。所有实验使用 **GPT-OSS-120B** 作为底层模型，系统提示和内存配置保持默认。
- **对比方法**：
  - 四种现有提示注入防御：**PIGuard**、**DataFilter**、**CommandSans**、**PromptArmor**。
  - 分别评估了**开箱即用（off-the-shelf）** 和 **微调/适配（retrained/adapted）** 版本下的真阳性率（TPR）和假阳性率（FPR）。
- **评估指标**：
  - ASR（Attack Success Rate）：攻击导致恶意指令写入持久化内存的比例。
  - RSR（Retrieval Success Rate）：已成功写入的恶意条目在后续会话中被检索并影响行为的比例。

## 6. 论文的主要结论与发现
- **内存投毒攻击是持久且有效的**：在所有攻击类别和Agent上，RSR均大于零，说明一次成功写入就能长期影响行为。
- **更积极读写内存的Agent更易被利用**：HERMES的平均ASR（66.67%）和RSR（64.70%）远高于OpenClaw（34.25%和17.40%），因为HERMES的写入策略更宽松、压缩阈值更低、检索自动注入系统提示。
- **弱信号攻击在宽松Agent中与强信号攻击有效性相当**：例如，Policy Conformant Fact Injection在HERMES上ASR达64.50%，而在OpenClaw仅8.33%。
- **现有提示注入防御无法充分覆盖内存投毒**：即使最强防御（PromptArmor）的开箱TPR也仅67.67%，且对弱信号攻击的检测率显著下降（42.50%），微调后改善有限。防御的根本盲区在于弱信号攻击在输入端不显示任何异常。
- **防御需要作用于写入路径而非输入边界**：作者建议收紧写入策略、实现源隔离、写路径溯源、压缩过滤和事中监控。

## 7. 优点
- **系统性**：首次将Agent内存系统抽象为写通道、漏洞、攻击三层，提供了统一的威胁模型和分类法。
- **实用性**：威胁模型贴近现实（黑盒外部攻击者），不要求特权访问或模型知识。
- **基准设计**：MPBench填补了现有基准（如LoCoMo、AgentDojo）只关注良性记忆或单会话提示注入的空白，采用两阶段评估（ASR+RSR）真实反映跨会话攻击。
- **防御分析深入**：揭示提示注入防御对弱信号攻击的失效原因（结构不同：弱信号攻击在输入端无模式），并指明正确防御方向。
- **实验覆盖全面**：大规模数据集、两种代表性的Agent系统、多种防御的对比，结果可靠。
